📊 打包脚本说明
================================================================================

【build.py】- 完整版打包脚本
  优点: 包含所有功能和资源
  缺点: 打包时间较长（5分钟+），文件较大（300MB+）
  用途: 完整功能的生产版本
  
  运行: python build.py

【build_simple.py】- 简化版打包脚本（推荐）
  优点: 打包时间短（2-3分钟），文件较小（100-150MB）
  缺点: 排除了一些重型AI库
  用途: 快速测试、基础功能演示
  
  运行: python build_simple.py

【直接PyInstaller命令】
  优点: 最直接、最灵活
  缺点: 需要手动配置参数
  用途: 自定义打包
  
  运行: 见"方式2"

🎯 常见问题解决
================================================================================

【问题1】PyInstaller 安装失败

  解决方案：
  # 方法A：使用清华源
  pip install -i https://pypi.tsinghua.edu.cn/simple pyinstaller
  
  # 方法B：升级pip后重试
  pip install --upgrade pip
  pip install pyinstaller
  
  # 方法C：使用完整路径
  python -m pip install pyinstaller

【问题2】打包过程中出现错误

  常见错误及解决方案：
  
  • ModuleNotFoundError: No module named 'xxx'
    → 确保所有依赖都已安装
    → pip install -r requirements.txt
  
  • 找不到资源文件
    → 检查 resources 和 config 目录存在
    → 确认 --add-data 参数正确
  
  • 权限错误
    → 使用管理员运行命令行
    → 或更改输出目录权限

【问题3】生成的 .exe 文件无法运行

  排查步骤：
  1. 检查是否为打包版本生成了所有文件
  2. 尝试从命令行运行查看错误信息
  3. 确认 _internal 文件夹完整
  4. 检查操作系统是否支持（需要 Windows 7+）

【问题4】文件过大（300MB+）

  优化方案：
  • 使用 build_simple.py（排除重型库）
  • 手动排除不需要的模块
  • 使用 UPX 压缩工具
  • 分离 AI 模型文件

📦 分发方案
================================================================================

打包完成后，有三种分发方式：

【方案1】直接分发文件夹
  
  步骤：
  1. 复制 dist 文件夹
  2. 重命名为 证件照采集系统_v1.0
  3. 直接发送给用户
  
  优点：最简单
  缺点：文件较大

【方案2】压缩分发（推荐）
  
  步骤：
  1. 压缩 dist 文件夹为 ZIP
     7z a 证件照采集系统_v1.0.7z dist\
     或
     Compress-Archive -Path dist -DestinationPath 证件照采集系统_v1.0.zip
  
  2. 发送压缩文件给用户
  
  用户步骤：
  1. 解压文件
  2. 双击 启动.bat 或 .exe
  
  优点：文件较小，易分发
  缺点：需要解压

【方案3】安装程序（最专业）
  
  步骤：
  1. 安装 NSIS：https://nsis.sourceforge.io/
  2. 使用 installer.nsi（见打包部署指南.md）
  3. 生成 .exe 安装程序
  
  优点：专业、支持卸载
  缺点：需要额外工具
