## 排查步骤

### 问题：选择图像时提示"用户不属于采集任务"

**步骤1：检查采集任务是否正确选择**
- 查看控制台输出中的 `[DEBUG] 采集任务切换` 信息
- 确认 `collection_id` 不为 None
- 确认采集任务名称正确

**步骤2：检查采集任务中是否有该用户**
- 查看 `[DEBUG] 采集任务中的用户数` 信息
- 查看列出的所有用户的身份证号
- 确认你要加载的图片对应的身份证号在列表中

**步骤3：检查图片文件名**
- 确保文件名格式为 `{身份证号}_{时间戳}.jpg`
- 确保身份证号是18位数字
- 查看控制台中的 `[DEBUG] 查询身份证号` 信息，确认提取的身份证号正确

**步骤4：检查数据库中的用户**
- 查看 `[DEBUG] 用户找到` 信息
- 如果显示 `None`，说明用户不在数据库中
- 需要在用户管理中添加该用户

### 问题：采集任务中没有显示用户

**可能原因1：采集任务没有users属性**
```
[DEBUG] 采集任务没有users属性
```
- 这表示数据库模型有问题
- 检查Collection模型是否正确定义了users关系

**可能原因2：采集任务为None**
```
[WARNING] 采集任务不存在: 5
```
- 采集任务ID不存在
- 检查数据库中是否有该采集任务

## 完整的调试流程

1. **启动应用**
   - 打开控制台，查看初始化信息

2. **选择采集任务**
   - 在下拉框中选择一个采集任务
   - 查看控制台输出，确认采集任务信息和用户列表

3. **加载图像**
   - 选择一个图片文件
   - 查看控制台输出，追踪用户识别过程
   - 如果有错误，查看错误信息

4. **分析问题**
   - 根据控制台输出判断问题所在
   - 参考上面的"排查步骤"

## 提交问题时需要提供的信息

如果问题仍未解决，请提供以下信息：

1. **采集任务选择时的控制台输出**
   - 复制 `[DEBUG] 采集任务切换` 到 `[DEBUG] 采集任务中的用户数` 之间的所有输出

2. **加载图像时的控制台输出**
   - 复制 `[DEBUG] 查询身份证号` 到 `[INFO] 用户识别成功` 或错误信息之间的所有输出

3. **图片文件名**
   - 提供你尝试加载的图片文件名

4. **采集任务名称和ID**
   - 提供你选择的采集任务名称

5. **用户信息**
   - 提供用户名和身份证号
