# 证件照采集系统 - 四种状态说明

## 状态定义

系统采用三个数据库状态值（pending、processing、completed），显示时呈现四种状态（无记录、待采集、待处理、已完成）。

### 1. 无记录（No Record）
- **定义**：用户信息存在于数据库，但没有采集记录
- **业务场景**：用户信息已导入系统，但还没有进行身份证读取
- **数据特征**：
  - 有用户记录（User表）
  - 无采集记录（CollectionRecord表）
- **显示**：无记录（灰色）
- **判断逻辑**：`if not records`

### 2. 待采集（Pending）
- **定义**：已读取身份证，但还没有拍照
- **业务场景**：用户已将身份证放在读卡器上，系统读取了信息并创建了采集记录，但用户还没有拍照
- **数据特征**：
  - 有用户记录（User表）
  - 有采集记录，状态为 `pending`（CollectionRecord表）
- **显示**：待采集（浅黄色）
- **判断逻辑**：`records[-1].status == 'pending'`
- **操作流程**：
  ```
  身份证读取 → 创建用户 → 创建采集记录(status='pending')
  ```

### 3. 待处理（Processing）
- **定义**：已拍照，但还没有进行图像处理
- **业务场景**：用户已完成拍照，系统保存了原始照片，但还没有进行背景替换、美颜等处理
- **数据特征**：
  - 有用户记录（User表）
  - 有采集记录，状态为 `processing`（CollectionRecord表）
  - 有原始照片（Photo表，photo_type='raw'）
- **显示**：待处理（浅橙色）
- **判断逻辑**：`records[-1].status == 'processing'`
- **操作流程**：
  ```
  拍照 → 保存原始照片 → 更新采集记录(status='processing')
  ```

### 4. 已完成（Completed）
- **定义**：已完成拍照和图像处理
- **业务场景**：用户已完成整个采集流程，原始照片和处理后照片都已保存
- **数据特征**：
  - 有用户记录（User表）
  - 有采集记录，状态为 `completed`（CollectionRecord表）
  - 有原始照片（Photo表，photo_type='raw'）
  - 有处理后照片（Photo表，photo_type='processed'）
- **显示**：已完成（浅绿色）
- **判断逻辑**：`records[-1].status == 'completed'`
- **操作流程**：
  ```
  图像处理 → 保存处理后照片 → 更新采集记录(status='completed')
  ```

## 状态流转

```
[用户导入] → 无记录（没有采集记录）
    ↓
[读取身份证] → 待采集 (pending)
    ↓
[拍照] → 待处理 (processing)
    ↓
[图像处理] → 已完成 (completed)
```

## 数据库状态 vs 显示状态

| 显示状态 | 判断逻辑 | 数据库status | 说明 |
|---------|---------|-------------|------|
| 无记录 | `if not records` | 无记录 | 没有采集记录 |
| 待采集 | `status == 'pending'` | pending | 已读身份证，未拍照 |
| 待处理 | `status == 'processing'` | processing | 已拍照，未处理 |
| 已完成 | `status == 'completed'` | completed | 已拍照并处理 |

**关键点**：
- 数据库中有三种状态值：`pending`、`processing`、`completed`
- "无记录"不是数据库状态，而是"没有采集记录"的状态
- 显示状态直接根据数据库status字段判断，不需要查询照片表
- 性能高效：只需查询CollectionRecord表，不需要JOIN Photo表

## 统计报表显示

### 统计信息标签
- **用户总数**：所有用户数量
- **照片总数**：所有照片数量
- **已完成**：状态为 `completed` 的采集记录数
- **待处理**：状态为 `processing` 的采集记录数
- **待采集**：状态为 `pending` 的采集记录数
- **完成率**：已完成数 / 总采集记录数 × 100%

### 完成率计算
完成率 = 已完成的采集记录数 / 总采集记录数 × 100%

**注意**：
- 按采集记录统计，不是按用户统计
- 总采集记录数 = completed + processing + pending
- 不包括"无记录"的用户（因为他们没有采集记录）

### 状态分布图
环形图显示三种状态的分布：
- **已完成**（绿色）：completed 记录数
- **待处理**（橙色）：processing 记录数
- **待采集**（黄色）：pending 记录数

### 完成率趋势图
折线图显示每日完成率变化：
- X轴：日期
- Y轴：完成率（0-100%）
- 计算方式：每日 completed 记录数 / 每日总记录数 × 100%

## 注意事项

1. **状态只能向前流转**：无记录 → 待采集 → 待处理 → 已完成
2. **不支持回退**：一旦进入下一个状态，不会自动回退到上一个状态
3. **无失败状态**：系统不记录失败状态，如果操作失败，用户可以重新操作
4. **状态判断依据**：
   - 显示层面：先判断有无记录，再根据status字段判断
   - 数据库层面：只有三个状态值（pending、processing、completed）
5. **性能优化**：显示状态时不查询照片表，只查询采集记录表
6. **数据一致性**：以数据库状态为准，不考虑照片文件被手动删除的情况
7. **界面显示**：所有状态显示均不使用emoji图标，只显示文字

## 批量处理

批量处理时，系统会：
1. 从文件名提取身份证号
2. 查询对应的用户
3. 保存处理后照片
4. 更新采集记录状态为 `completed`

如果文件名格式不正确或用户不存在，只保存文件，不更新数据库。
