# 批量处理照片记录问题修复

## 问题描述

批量处理后发现：

1. **统计报表页面**：待处理数量正确更新（状态变为"已完成"）✓
2. **数据管理页面**：
   - 用户的"处理照片"数量仍显示0 ✗
   - 统计信息里的"照片总数"没有变化 ✗

## 问题原因

在之前的修复中，虽然添加了数据库状态更新功能，但是在多规格处理时，照片记录的添加逻辑有问题：

```python
# 原来的代码
if os.path.isfile(output_path):
    # 单个文件 - 添加照片记录 ✓
    photo = db.add_photo(...)
else:
    # 多规格处理，output_path是目录
    print(f"[INFO] 多规格处理，跳过单个照片记录添加")  # ✗ 这里跳过了！
```

### 为什么会跳过？

- **单规格处理**：`output_path` 是具体的文件路径（如 `output/user_19.jpg`）
- **多规格处理**：`output_path` 是输出目录路径（如 `output/`）

原来的逻辑认为多规格处理会生成多个文件，所以跳过了照片记录添加。但这导致：
- 数据库中没有照片记录
- 数据管理页面显示"处理照片: 0"
- 统计信息中的照片总数不准确

## 解决方案

修改 `_update_database_status()` 方法，在多规格处理时扫描输出目录，为每个生成的文件添加照片记录。

### 修改后的逻辑

```python
# 添加照片记录
if os.path.isfile(output_path):
    # 单规格处理：单个文件
    file_size = os.path.getsize(output_path)
    photo = db.add_photo(
        user_id=user_id,
        photo_type='processed',
        file_path=output_path,
        file_size=file_size
    )
    print(f"[INFO] 添加照片记录: photo_id={photo.id}")
    
elif os.path.isdir(output_path):
    # 多规格处理：扫描输出目录，添加所有相关照片
    print(f"[INFO] 多规格处理，扫描输出目录添加照片记录")
    
    # 从输入文件名提取基础名称
    base_name = os.path.splitext(os.path.basename(input_path))[0]
    
    # 扫描输出目录中所有相关文件
    added_count = 0
    for file in os.listdir(output_path):
        # 检查文件是否属于当前用户（文件名包含基础名称）
        if file.startswith(base_name) and file.lower().endswith(('.jpg', '.jpeg', '.png')):
            file_path = os.path.join(output_path, file)
            if os.path.isfile(file_path):
                file_size = os.path.getsize(file_path)
                photo = db.add_photo(
                    user_id=user_id,
                    photo_type='processed',
                    file_path=file_path,
                    file_size=file_size
                )
                added_count += 1
                print(f"[INFO] 添加照片记录: photo_id={photo.id}, file={file}")
    
    print(f"[INFO] 多规格处理完成，共添加 {added_count} 条照片记录")
```

## 工作原理

### 1. 识别处理类型

通过检查 `output_path` 的类型判断：

- `os.path.isfile(output_path)` → 单规格处理
- `os.path.isdir(output_path)` → 多规格处理

### 2. 单规格处理

直接添加单个照片记录：

```
输入: raw_19_20260425143026.jpg
输出: output/raw_19_20260425143026_1inch_white.jpg
    ↓
添加1条照片记录
```

### 3. 多规格处理

扫描输出目录，添加所有相关照片记录：

```
输入: raw_19_20260425143026.jpg
输出目录: output/
    ├── raw_19_20260425143026_20260425_150000_1inch_white.jpg
    ├── raw_19_20260425143026_20260425_150000_1inch_blue.jpg
    ├── raw_19_20260425143026_20260425_150000_2inch_white.jpg
    └── raw_19_20260425143026_20260425_150000_2inch_blue.jpg
    ↓
扫描目录，找到所有以 "raw_19_20260425143026" 开头的文件
    ↓
添加4条照片记录
```

### 4. 文件匹配逻辑

```python
# 提取基础名称
base_name = os.path.splitext(os.path.basename(input_path))[0]
# 例如: "raw_19_20260425143026"

# 匹配规则

    # 匹配成功，添加照片记录
```

## 注意事项

### 1. 文件名匹配

多规格处理时，通过文件名前缀匹配来识别属于同一用户的照片：

```python
base_name = "raw_19_20260425143026"  # 从输入文件提取

# 匹配这些文件:
"raw_19_20260425143026_timestamp_1inch_white.jpg"  ✓
"raw_19_20260425143026_timestamp_2inch_blue.jpg"   ✓

# 不匹配这些文件:
"raw_27_20260425143030_timestamp_1inch_white.jpg"  ✗
"other_file.jpg"                                    ✗
```

### 2. 文件扩展名

支持的图像格式：`.jpg`、`.jpeg`、`.png`

### 3. 输出目录

确保输出目录中只包含当前批次的处理结果，避免误添加其他文件的记录。

### 4. 性能考虑

多规格处理时，如果生成大量文件（如10个规格 × 5个颜色 = 50个文件），扫描和添加记录可能需要几秒钟。

## 常见问题

### Q1: 为什么照片数量还是0？

A: 可能的原因：
1. 数据管理页面没有刷新（点击"刷新"按钮）
2. 文件名格式不正确，无法识别用户
3. 输出目录路径有问题

### Q2: 照片总数增加了，但用户的照片数量没变？

A: 检查照片记录的 `user_id` 是否正确。可能是文件名识别错误，照片记录添加到了错误的用户。

### Q3: 多规格处理后，照片数量不对？

A: 检查：
1. 输出目录中实际生成了多少文件
2. 控制台日志中显示添加了多少条记录
3. 文件名是否都以正确的前缀开头

### Q4: 单规格处理正常，多规格处理有问题？

A: 检查输出目录是否正确传递。多规格处理时传递的应该是目录路径，不是文件路径。
