# 采集进度趋势 - 三率对比说明

## 概述

采集进度趋势图显示三条趋势线，全面展示证件照采集流程的各个阶段进度：

1. **完成率**（绿色）- 已完成图像处理的用户占比
2. **处理率**（橙色）- 已拍照的用户占比
3. **采集率**（蓝色）- 已开始采集流程的用户占比

## 三率定义

### 1. 完成率（绿色线）

**定义**：截止到当天，已完成整个采集流程的用户占比

**计算公式**：
```
完成率 = 截止到当天累计已完成数 / 用户总数 × 100%
```

**业务含义**：
- 用户已完成：拍照 + 图像处理
- 状态为 `completed`
- 代表最终完成的工作量

### 2. 处理率（橙色线）

**定义**：截止到当天，至少已拍照的用户占比

**计算公式**：
```
处理率 = 截止到当天累计(已完成 + 待处理)数 / 用户总数 × 100%
```

**业务含义**：
- 用户已拍照，可能已处理或待处理
- 状态为 `completed` 或 `processing`
- 代表拍照环节的完成情况

### 3. 采集率（蓝色线）

**定义**：截止到当天，至少已开始采集流程的用户占比

**计算公式**：
```
采集率 = 截止到当天累计(已完成 + 待处理 + 待采集)数 / 用户总数 × 100%
```

**业务含义**：
- 用户已读取身份证，进入采集流程
- 状态为 `completed`、`processing` 或 `pending`
- 代表整体采集覆盖率

## 三率关系

### 数学关系

```
采集率 ≥ 处理率 ≥ 完成率
```

这是因为：
- 采集率包含：已完成 + 待处理 + 待采集
- 处理率包含：已完成 + 待处理
- 完成率包含：已完成

### 业务含义

**三条线之间的间隙代表各阶段的"积压"**：

1. **采集率 - 处理率 = 待采集率**
   - 已读身份证但还没拍照的用户占比
   - 间隙越大，说明拍照环节积压越多

2. **处理率 - 完成率 = 待处理率**
   - 已拍照但还没处理的用户占比
   - 间隙越大，说明图像处理环节积压越多

3. **100% - 采集率 = 无记录率**
   - 还没开始采集流程的用户占比
   - 间隙越大，说明还有很多用户没开始

## 实际数据示例

假设有100个用户，某天的数据：

| 状态 | 人数 | 累计人数 | 对应的率 |
|------|------|---------|---------|
| 无记录 | 20 | - | - |
| 待采集 | 15 | 80 | 采集率 80% |
| 待处理 | 25 | 65 | 处理率 65% |
| 已完成 | 40 | 40 | 完成率 40% |
| **总计** | **100** | - | - |

**计算**：
- 完成率 = 40 / 100 = 40%
- 处理率 = (40 + 25) / 100 = 65%
- 采集率 = (40 + 25 + 15) / 100 = 80%

**间隙分析**：
- 待采集率 = 80% - 65% = 15%（15个用户）
- 待处理率 = 65% - 40% = 25%（25个用户）
- 无记录率 = 100% - 80% = 20%（20个用户）

## 使用场景

### 1. 监控整体进度

通过三条线的位置，快速了解整体采集进度：
- 三条线都很高（接近100%）→ 工作接近完成
- 三条线都很低 → 工作刚开始
- 三条线间隙大 → 各环节积压严重

### 2. 发现瓶颈

通过观察间隙大小，找出瓶颈环节：
- 蓝橙间隙大 → 拍照环节是瓶颈，需要加快拍照
- 橙绿间隙大 → 图像处理环节是瓶颈，需要加快处理
- 顶部间隙大 → 身份证读取环节是瓶颈，需要加快录入

### 3. 评估工作效率

通过观察线的斜率，评估各环节效率：
- 斜率陡 → 该阶段推进快
- 斜率平 → 该阶段推进慢
- 斜率为0 → 该阶段停滞

### 4. 预测完成时间

根据完成率的增长速度，预测整体完成时间：
- 如果完成率每天增长2%，还需50天完成
- 如果完成率增长加速，可以提前完成
- 如果完成率增长减速，需要调整计划

## 代码实现

### 数据查询

```python
# 基础查询：截止到当天的所有记录
base_query = db.db.query(CollectionRecord).filter(
    CollectionRecord.collection_date <= current_date
)

# 1. 完成率：只统计已完成
completed_query = base_query.filter(CollectionRecord.status == 'completed')
cumulative_completed = completed_query.count()
completion_rate = (cumulative_completed / total_users * 100)

# 2. 处理率：统计已完成 + 待处理
processed_query = base_query.filter(
    CollectionRecord.status.in_(['completed', 'processing'])
)
cumulative_processed = processed_query.count()
processing_rate = (cumulative_processed / total_users * 100)

# 3. 采集率：统计已完成 + 待处理 + 待采集
collected_query = base_query.filter(
    CollectionRecord.status.in_(['completed', 'processing', 'pending'])
)
cumulative_collected = collected_query.count()
collection_rate = (cumulative_collected / total_users * 100)
```

## 注意事项

1. **三率都是累计值**：都是截止到当天的累计数据，所以都是递增的（或持平）

2. **分母固定**：三个率的分母都是用户总数，保持一致

3. **数据一致性**：三条线的数据来源相同，确保一致性

4. **性能优化**：使用数据库查询而不是遍历，提高性能

5. **异常处理**：如果查询失败，三个率都设为0，避免图表崩溃

## 与其他图表的关系

### 与状态分布图的关系

状态分布图（环形图）显示的是当前时刻的状态分布，而采集进度趋势图显示的是随时间变化的累计进度。

**互补关系**：
- 状态分布图：当前快照
- 采集进度趋势图：历史变化

### 与统计信息的关系

统计信息中的完成率是整个时间段的总体完成率，等于采集进度趋势图中最后一天的完成率。

**一致性**：
- 统计信息完成率 = 趋势图最后一天的完成率
