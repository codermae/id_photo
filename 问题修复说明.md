# 问题修复说明

## 修复的问题

### 1. 数据管理页面状态筛选失败

**问题描述**：
在数据管理页面通过状态进行筛选时，选择"已完成"和"待处理"都提示"未找到匹配用户"。

**问题原因**：
`data_view.py` 中的 `apply_filters` 方法使用的状态映射还是旧的两状态系统：
```python
status_map = {
    '已采集': 'completed',
    '待采集': 'pending',
    '失败': 'failed'
}
```

**修复方案**：
更新状态映射为新的三状态系统：
```python
status_map = {
    '已完成': 'completed',
    '待处理': 'processing',
    '待采集': 'pending'
}
```

**修改文件**：
- `id_photo_system/views/data_view.py` - 第437-441行

---

### 2. 统计报表缺少"无记录"统计

**问题描述**：
统计报表的统计信息和采集状态分布图中没有显示"无记录"的用户数量。

**问题原因**：
系统只统计了有采集记录的用户（pending、processing、completed），没有统计只有用户信息但没有采集记录的用户。

**修复方案**：

#### 2.1 计算无记录用户数
在 `report_view.py` 的 `DataWorker.run()` 方法中添加：
```python
# 计算无记录用户数（有用户信息但没有采集记录）
no_record_count = 0
for user in users:
    records = self.db.get_records_by_user(user.id)
    if not records:
        no_record_count += 1

stats = {
    ...
    'no_record': no_record_count,
    ...
}
```

#### 2.2 添加界面标签
在统计信息区域添加第6个标签：
```python
self.l6 = QLabel("无记录: 0")
self.l6.setStyleSheet("color: gray; font-weight: bold;")
```

原来的"完成率"标签改为 `self.l7`。

#### 2.3 更新界面显示
在 `update_ui` 方法中添加：
```python
self.l6.setText(f"无记录: {s['no_record']}")
```

#### 2.4 更新状态分布图
在 `echarts_generator.py` 的 `generate_collection_status_chart` 方法中添加：
```python
if stats.get('no_record', 0) > 0:
    data.append({
        'value': stats['no_record'], 
        'name': '无记录', 
        'itemStyle': {'color': '#909399'}  # 灰色
    })
```

**修改文件**：
- `id_photo_system/views/report_view.py` - 第21-40行、第163-177行、第268-275行
- `id_photo_system/controllers/echarts_generator.py` - 第11-28行

### 3. 完成率趋势计算逻辑说明

**问题描述**：
用户询问完成率趋势是按照日期每天算的吗？具体计算逻辑是什么？

**回答**：

#### 计算方式

完成率趋势图是按日期逐天计算的，具体逻辑如下：

1. **遍历日期范围**：从开始日期到结束日期，逐天计算
2. **获取当天记录**：获取 `collection_date` 等于当天的所有采集记录
3. **计算完成率**：
   ```python
   completed = len([r for r in records if r.status == 'completed'])
   total = len(records)  # 包括 pending、processing、completed
   rate = (completed / total * 100) if total > 0 else 0
   ```

#### 完成率定义

**完成率 = 当天已完成的采集记录数 / 当天总采集记录数 × 100%**

#### 关键点

- **按采集日期统计**：只统计 `collection_date` 等于当天的记录
- **只统计有记录的用户**：无记录的用户不参与计算
- **总记录数包含所有状态**：pending + processing + completed
- **已完成数**：只统计 `status == 'completed'` 的记录

#### 示例

假设 2026-04-25 这天有5条采集记录：
- 3条 completed（已完成）
- 1条 processing（待处理）
- 1条 pending（待采集）

完成率 = 3 / 5 × 100% = 60%

#### 与统计信息的区别

- **统计信息中的完成率**：整个日期范围内的总体完成率（一个数值）
- **趋势图中的完成率**：每天的完成率（多个数值，形成曲线）
