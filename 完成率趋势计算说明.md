# 完成率趋势图计算逻辑说明

## 概述

完成率趋势图是一个折线图，显示在指定日期范围内每天的完成率变化情况。

## 计算方式（2026-04-25 更新）

### 完成率定义

**完成率 = 截止到当天累计已完成数 / 用户总数 × 100%**

- **分子**：截止到当天累计已完成的采集记录数（累计值）
- **分母**：用户总数（包含无记录、待采集、待处理、已完成）
- **特点**：完成率应该是递增的（或持平），不会下降
- **目的**：显示整体工作进度的累计变化趋势

### 1. 获取用户总数

在开始计算前，先获取用户总数（包含无记录用户）：

```python
# 获取用户总数（包含无记录用户）
total_users = db.get_user_count(collection_id)
```

### 2. 按日期遍历

从开始日期到结束日期，逐天计算完成率：

```python
current_date = start_date
while current_date <= end_date:
    # 计算当天的完成率
    current_date = current_date + timedelta(days=1)
```

### 3. 获取当天的采集记录

对于每一天，获取截止到该日期的所有已完成记录（累计）：

```python
# 获取截止到当天的所有已完成记录（累计）
query = db.db.query(CollectionRecord).filter(
    CollectionRecord.collection_date <= current_date,
    CollectionRecord.status == 'completed'
)
cumulative_completed = query.count()
```

**注意**：使用 `<=` 而不是 `==`，这样可以获取累计数量。

### 4. 计算当天完成率

```python
# 累计已完成数
cumulative_completed = query.count()

# 累计完成率 = 累计已完成数 / 用户总数
rate = (cumulative_completed / total_users * 100) if total_users > 0 else 0
```

## 关键变化

### 旧计算方式（已废弃）

```python
# 完成率 = 已完成数 / 当天采集记录数 × 100%
total = len(records)
rate = (completed / total * 100) if total > 0 else 0
```

**问题**：
- 不包含无记录用户
- 每天的分母不同（当天采集记录数）
- 不能真实反映整体进度

### 新计算方式（当前）

```python
# 完成率 = 已完成数 / 用户总数 × 100%
total_users = db.get_user_count(collection_id)
rate = (completed / total_users * 100) if total_users > 0 else 0
```

**优点**：
- 包含无记录用户
- 分母固定（用户总数）
- 真实反映整体工作进度
- 与统计信息中的完成率计算方式一致

## 示例

假设有以下数据：

**用户总数**：100人

**采集记录时间线**：

| 日期 | 当天完成数 | 截止当天累计完成数 | 累计完成率 |
|------|-----------|------------------|-----------|
| 04-20 | 5 | 5 | 5% |
| 04-21 | 3 | 8 | 8% |
| 04-22 | 0 | 8 | 8% |
| 04-23 | 10 | 18 | 18% |
| 04-24 | 7 | 25 | 25% |
| 04-25 | 15 | 40 | 40% |

**计算说明**：
- 04-20：当天完成5个，累计5个，完成率 = 5/100 = 5%
- 04-21：当天完成3个，累计8个，完成率 = 8/100 = 8%
- 04-22：当天完成0个，累计8个，完成率 = 8/100 = 8%（持平）
- 04-23：当天完成10个，累计18个，完成率 = 18/100 = 18%
- ...

**趋势特点**：
- 完成率是递增的（或持平）
- 不会出现下降
- 最终会达到总体完成率

## 图表显示

- **X轴**：日期（格式：MM-DD，如 04-25）
- **Y轴**：完成率（0-100%）
- **数据点**：每天的完成率
- **曲线**：平滑曲线连接各数据点
- **区域填充**：曲线下方有半透明填充

## 与统计信息的关系

### 统计信息中的完成率

统计信息中显示的完成率是整个日期范围内的总体完成率：

```python
stats_data = db.get_collection_stats(start_date, end_date, collection_id)
completion_rate = stats_data['completion_rate']
```

计算方式：
```python
# 获取日期范围内的所有记录
records = query.filter(
    CollectionRecord.collection_date >= start_date,
    CollectionRecord.collection_date <= end_date
).all()

# 获取用户总数（包含无记录用户）
total_users = self.get_user_count(collection_id)

completed = len([r for r in records if r.status == 'completed'])
completion_rate = (completed / total_users * 100) if total_users > 0 else 0
```

### 趋势图中的完成率

趋势图显示的是每天的累计完成率，是一个时间序列数据。

**一致性**：
- 统计信息：整个时间段的总体完成率（一个数值）
- 趋势图：每天的累计完成率（多个数值，形成递增曲线）
- **计算方式相同**：都是 已完成数 / 用户总数 × 100%
- **区别**：趋势图是累计值，统计信息是最终值

## 使用场景

1. **查看采集进度**：了解每天的完成情况
2. **发现异常**：某天完成率突然下降可能表示出现问题
3. **评估效率**：对比不同日期的完成率，评估工作效率
4. **预测完成时间**：根据趋势预测整体完成时间

## 注意事项

1. **日期范围**：默认显示最近30天，可以通过日期选择器调整
2. **数据缩放**：图表支持鼠标滚轮缩放，方便查看细节
3. **空数据处理**：如果某天没有采集记录，完成率显示为0
4. **实时更新**：点击"刷新统计"按钮可以更新最新数据
5. **包含无记录用户**：完成率计算包含所有导入的用户，即使他们还没有开始采集

### echarts_generator.py

```python
@staticmethod
def generate_success_rate_trend_chart(start_date, end_date, db, collection_id):
    """生成完成率趋势折线图
    
    完成率 = 截止到当天累计已完成数 / 用户总数 × 100%
    （累计完成率，应该是递增的）
    """
    dates = []
    completion_rates = []
    
    # 获取用户总数（包含无记录用户）
    total_users = db.get_user_count(collection_id)
    
    current_date = start_date
    while current_date <= end_date:
        dates.append(current_date.strftime('%m-%d'))
        
        try:
            # 获取截止到当天的所有已完成记录（累计）
            query = db.db.query(CollectionRecord).filter(
                CollectionRecord.collection_date <= current_date,
                CollectionRecord.status == 'completed'
            )
            
            if collection_id:
                query = query.filter(CollectionRecord.collection_id == collection_id)
            elif db.current_collection_id:
                query = query.filter(CollectionRecord.collection_id == db.current_collection_id)
            
            # 累计已完成数
            cumulative_completed = query.count()
            
            # 累计完成率 = 累计已完成数 / 用户总数
            rate = (cumulative_completed / total_users * 100) if total_users > 0 else 0
        except Exception as e:
            print(f"[ERROR] 计算完成率失败: {e}")
            rate = 0
        
        completion_rates.append(rate)
        current_date = current_date + timedelta(days=1)
    
    return {
        'title': {'text': '完成率趋势', 'left': 'center'},
        'tooltip': {
            'trigger': 'axis',
            'formatter': '{b}<br/>完成率: {c}%'
        },
        'xAxis': {'type': 'category', 'data': dates, 'boundaryGap': False},
        'yAxis': {
            'type': 'value', 
            'min': 0, 
            'max': 100,
            'axisLabel': {'formatter': '{value}%'}
        },
        'series': [{
            'name': '完成率',
            'type': 'line',
            'data': completion_rates,
            'smooth': True,
            'itemStyle': {'color': '#5470C6'},
            'areaStyle': {'color': 'rgba(84, 112, 198, 0.2)'}
        }]
    }
```

- **2026-04-25 v1**：修改为 完成率 = 当天已完成数 / 用户总数，包含无记录用户
