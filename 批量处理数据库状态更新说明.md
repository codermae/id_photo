# 批量处理数据库状态更新说明

## 问题描述

批量处理照片后，用户的状态仍然是"待处理"，没有自动更新为"已完成"。但是单张处理并保存后，状态会正确更新为"已完成"。

## 问题原因

批量处理器（`BatchProcessor`）原来只是保存了处理后的图像文件，但没有更新数据库中的采集记录状态。而单张处理在保存时会：

1. 添加照片记录到数据库
2. 更新采集记录状态为 `completed`

## 解决方案

在批量处理器中添加 `_update_database_status()` 方法，在保存图像成功后自动更新数据库状态。

### 修改内容

#### 1. 单规格处理

在 `_process_single_spec_file()` 方法中，保存成功后调用数据库更新：

```python
if success:
    # 更新处理结果
    item['status'] = 'completed'
    item['result_path'] = output_path
    item['processing_time'] = time.time() - start_time
    item['quality_score'] = processor.get_quality_score()
    item['file_size_after'] = os.path.getsize(output_path)
    
    # 更新数据库状态（新增）
    self._update_database_status(item['input_path'], output_path)
    
    self._log_status(f"处理完成: {item['input_path']}", "success")
    return True
```

#### 2. 多规格处理

在 `_process_multi_spec_file()` 方法中，处理成功后调用数据库更新：

## 更新逻辑详解

### 1. 用户识别

从输入文件名中提取用户信息，支持两种格式：

#### 格式1：测试数据格式
```
raw_{user_id}_{timestamp}.jpg
```

**示例**：`raw_19_20260425143026.jpg`

**提取逻辑**：
```python
if filename.startswith('raw_') and len(parts) >= 3:
    user_id = int(parts[1])  # 提取用户ID
```

#### 格式2：实际拍照格式
```
{id_number}_{timestamp}.jpg
```

**示例**：`110105199001011234_20260425143026.jpg`

**提取逻辑**：
```python
elif len(parts) >= 2:
    potential_id = parts[0]  # 提取身份证号
    if potential_id.isdigit() and len(potential_id) in [15, 18]:
        user = db.get_user_by_id_number(potential_id)
        user_id = user.id
```

### 2. 添加照片记录

#### 单规格处理

保存单个文件，添加一条照片记录：

```python
if os.path.isfile(output_path):
    file_size = os.path.getsize(output_path)
    photo = db.add_photo(
        user_id=user_id,
        photo_type='processed',
        file_path=output_path,
        file_size=file_size
    )
```

#### 多规格处理

生成多个文件到目录，跳过单个照片记录添加（因为有多个文件）：

```python
else:
    # 多规格处理，output_path是目录
    print(f"[INFO] 多规格处理，跳过单个照片记录添加")
```

### 3. 更新采集记录状态

#### 如果用户已有采集记录

更新最新的记录状态为 `completed`：

```python
existing_records = db.get_records_by_user(user_id)
if existing_records:
    latest_record = existing_records[-1]
    if latest_record.status != 'completed':
        latest_record.status = 'completed'
        latest_record.notes = f'批量处理完成: {os.path.basename(output_path)}'
        db.db.commit()
```

#### 如果用户没有采集记录

创建新的采集记录，状态为 `completed`：

```python
else:
    import getpass

    record = db.add_record(
        user_id=user_id,

        status='completed',

```

## 状态流转

### 正常流程

```
1. 用户导入 → 无记录
2. 读取身份证 → 待采集 (pending)
3. 拍照 → 待处理 (processing)
4. 批量处理 → 已完成 (completed)  ← 新增的自动更新
```

### 批量处理前后对比

#### 修改前

```
用户ID 19: 待处理 (processing)
用户ID 27: 待处理 (processing)
    ↓
批量处理（保存图像）
    ↓
用户ID 19: 待处理 (processing)  ← 状态未变
用户ID 27: 待处理 (processing)  ← 状态未变
```

#### 修改后

```
用户ID 19: 待处理 (processing)
用户ID 27: 待处理 (processing)
    ↓
批量处理（保存图像 + 更新数据库）
    ↓
用户ID 19: 已完成 (completed)  ← 自动更新
用户ID 27: 已完成 (completed)  ← 自动更新
```

## 注意事项

### 1. 文件名格式

确保输入文件名符合以下格式之一：

- `raw_{user_id}_{timestamp}.jpg`（测试数据）
- `{id_number}_{timestamp}.jpg`（实际拍照）

否则无法识别用户，状态不会更新。

### 2. 用户必须存在

文件名中的用户ID或身份证号必须在数据库中存在，否则无法更新状态。

### 3. 采集记录

- 如果用户已有采集记录，会更新最新的记录
- 如果用户没有采集记录，会创建新的记录（状态为 completed）

### 4. 多规格处理

多规格处理时，只更新采集记录状态，不添加单个照片记录（因为生成了多个文件）。

### 5. 错误处理

如果数据库更新失败，不会影响图像保存。图像文件仍然会保存成功，只是数据库状态不会更新。
