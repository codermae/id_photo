# 图像处理页面文件名格式说明

## 问题描述

在图像处理页面选择图像时，系统提示"文件名格式错误，无法识别身份证号"。

## 问题原因

系统生成的测试数据文件名格式为 `raw_{user_id}_{timestamp}.jpg`，但是图像处理页面原来只支持 `{id_number}_{timestamp}.jpg` 格式，导致无法识别测试数据的文件名。

## 解决方案

更新图像处理页面的文件名解析逻辑，支持两种文件名格式：

### 格式1：测试数据格式

```
raw_{user_id}_{timestamp}.jpg
```

**示例**：
- `raw_1_20260425143026.jpg`
- `raw_25_20260425150000.jpg`

**解析逻辑**：
1. 检查文件名是否以 `raw_` 开头
2. 提取第二部分作为用户ID（整数）
3. 通过用户ID查询数据库
4. 验证用户是否属于当前选定的采集任务

### 格式2：实际拍照格式

```
{id_number}_{timestamp}.jpg
```

**示例**：
- `110105199001011234_20260425143026.jpg`
- `320102198512125678_20260425150000.jpg`

**解析逻辑**：
1. 提取第一部分作为身份证号
2. 验证是否为有效的身份证号（15或18位数字）
3. 通过身份证号和采集任务ID查询数据库
4. 验证用户是否存在于当前采集任务中

## 使用说明

### 1. 选择采集任务

在图像处理页面顶部，先选择正确的采集任务。这很重要，因为系统会验证图像对应的用户是否属于该采集任务。

### 2. 选择图像

点击"选择图像"按钮，选择要处理的原始照片。

### 3. 文件名验证

系统会自动识别文件名格式：

- **测试数据**：如果文件名以 `raw_` 开头，系统会提取用户ID
- **实际照片**：如果文件名以身份证号开头，系统会提取身份证号

### 4. 用户验证

系统会验证：
- 用户是否存在于数据库中
- 用户是否属于当前选定的采集任务

如果验证失败，会显示相应的错误提示。

## 错误提示说明

### 1. 用户不属于当前采集任务

```
用户 XXX 不属于选定的采集任务

请选择正确的采集任务
```

**解决方法**：
- 在页面顶部切换到正确的采集任务
- 或者选择"全部采集任务"

### 2. 用户ID不存在

```
用户ID XXX 不存在数据库中
```

**解决方法**：
- 检查文件名是否正确
- 确认该用户是否已添加到系统中

### 3. 身份证号不存在

```
身份证号 XXX 在选定的采集任务中不存在

请检查：
1. 是否选择了正确的采集任务
2. 用户是否已添加到该采集任务
```

**解决方法**：
- 切换到正确的采集任务
- 确认该用户是否已添加到系统中

### 4. 文件名格式错误

```
文件名格式错误

支持的格式：
1. raw_{用户ID}_{时间戳}.jpg
2. {身份证号}_{时间戳}.jpg
```

**解决方法**：
- 检查文件名格式是否正确
- 确保文件名符合上述两种格式之一

## 文件名生成规则

### 测试数据生成

测试数据生成脚本 (`generate_test_data.py`) 使用以下格式：

```python
# 原始照片
raw_filename = f"raw_{user.id}_{datetime.now().strftime('%Y%m%d%H%M%S')}.jpg"

# 处理后照片
processed_filename = f"processed_{user.id}_{datetime.now().strftime('%Y%m%d%H%M%S')}.jpg"
```

### 实际拍照

摄像头拍照模块应该使用身份证号作为文件名前缀：

```python
filename = f"{user.id_number}_{datetime.now().strftime('%Y%m%d%H%M%S')}.jpg"
```

## 技术细节

### 文件名解析流程

```python
# 1. 获取文件名
filename = os.path.basename(filepath)
parts = filename.split('_')

# 2. 判断格式
if filename.startswith('raw_') and len(parts) >= 3:
    # 格式1: raw_{user_id}_{timestamp}.jpg
    user_id = int(parts[1])
    user = db.get_user_by_id(user_id)
    
elif len(parts) >= 2:
    # 格式2: {id_number}_{timestamp}.jpg
    id_number = parts[0]
    if id_number.isdigit() and len(id_number) in [15, 18]:
        user = db.get_user_by_id_number(id_number, collection_id)
```

### 用户验证

```python
# 验证用户是否属于当前采集任务
if self.current_collection_id is None or user.collection_id == self.current_collection_id:
    # 验证通过
    self.current_user_id = user.id
else:
    # 验证失败
    QMessageBox.warning(...)
```

## 最佳实践

### 1. 文件命名规范

- **测试环境**：使用 `raw_{user_id}_{timestamp}.jpg` 格式
- **生产环境**：使用 `{id_number}_{timestamp}.jpg` 格式

### 2. 采集任务选择

- 处理图像前，先选择正确的采集任务
- 如果不确定，可以选择"全部采集任务"

### 3. 批量处理

批量处理时，确保所有文件名格式一致，并且用户都属于同一个采集任务。

### 4. 文件组织

建议按采集任务组织文件：

```
data/photos/raw/
  ├── collection_1/
  │   ├── raw_1_20260425143026.jpg
  │   ├── raw_2_20260425143030.jpg
  │   └── ...
  └── collection_2/
      ├── raw_50_20260425150000.jpg
      └── ...
```

## 注意事项

1. **文件名不能包含空格**：使用下划线分隔各部分
2. **时间戳格式固定**：YYYYMMDDHHmmss（14位数字）
3. **身份证号必须有效**：15位或18位数字
4. **用户ID必须存在**：必须是数据库中已有的用户ID
5. **采集任务必须匹配**：用户必须属于选定的采集任务
