# AI算法实现

## 模块概述

AI算法实现模块集成了多个深度学习和计算机视觉算法，包括人脸检测、人脸识别、图像分割等功能。通过使用业界先进的模型和库，实现了高精度的人脸处理和图像理解能力。

**核心算法**：
- 人脸检测：fdlite (TensorFlow Lite) + OpenCV级联分类器
- 人脸识别：face-recognition库 (基于dlib)
- 人像分割：rembg (U2-Net深度学习模型)
- 人脸关键点检测：MediaPipe
- 质量评估：多维度综合评分系统

## 功能特性

### 1. 人脸检测 (Face Detection)

#### 检测方法

**优先级顺序**：

1. **fdlite (TensorFlow Lite)** - 推荐
   - 模型：SSD MobileNet
   - 精度：高
   - 速度：快（GPU加速）
   - 优点：准确率高，速度快，支持多人脸检测

2. **OpenCV级联分类器** - 备选
   - 模型：LBP Cascade / Haar Cascade
   - 精度：中等
   - 速度：快
   - 优点：无需额外依赖，轻量级

#### 检测稳定性机制

```python
# 检测历史记录
detection_history = []  # 存储最近5次检测结果
history_size = 5
stable_threshold = 3    # 5次中至少3次检测到才认为稳定

# 稳定性判断
if len(detection_history) >= history_size:
    stable_count = sum(1 for d in detection_history if d is not None)
    is_stable = stable_count >= stable_threshold
```

#### 检测参数

| 参数 | 值 | 说明 |
|-----|-----|------|
| 置信度阈值 | 0.3 | 检测置信度最小值 |
| 最小人脸大小 | 20×20 | 最小检测人脸尺寸 |
| 最大人脸大小 | 图像宽度 | 最大检测人脸尺寸 |
| 检测间隔 | 每帧 | 检测频率 |

### 2. 人脸识别 (Face Recognition)

#### 识别原理

```
原始人脸图像
    ↓
人脸检测和对齐
    ↓
特征提取 (128维向量)
    ↓
特征比对 (欧氏距离)
    ↓
相似度计算 (0-100%)
```

#### 特征提取

- **模型**：ResNet-34 (基于dlib)
- **特征维度**：128维
- **提取时间**：~100ms/人脸
- **存储格式**：二进制 (16字节 × 128 = 2048字节)

#### 相似度计算

```python
# 欧氏距离计算
distance = np.linalg.norm(encoding1 - encoding2)

# 相似度转换
similarity = max(0, 100 - distance * 50)  # 0-100%

# 判断阈值
RECOGNITION_THRESHOLD = 0.6  # 距离阈值
if distance < RECOGNITION_THRESHOLD:
    # 识别成功
else:
    # 识别失败
```

#### 防重复采集

- 每个用户保存一个人脸特征向量
- 新采集的照片与已有特征进行比对
- 相似度超过阈值则判定为重复
- 防止同一用户多次采集

### 3. 人像分割 (Portrait Segmentation)

#### 分割算法

**优先级顺序**：

1. **rembg (U2-Net)** - 推荐
   - 模型：U2-Net深度学习模型
   - 精度：非常高
   - 速度：1-2秒/图像
   - 优点：边缘精细，处理复杂背景

2. **高级肤色检测** - 备选
   - 方法：HSV颜色空间肤色检测
   - 精度：中等
   - 速度：快
   - 优点：无需深度学习，轻量级

3. **基础肤色检测** - 备用
   - 方法：简单肤色范围检测
   - 精度：低
   - 速度：很快
   - 优点：最轻量级

#### 分割流程

```python
# U2-Net分割流程
1. 图像预处理 → 归一化、缩放
2. 模型推理 → 获取分割掩码
3. 掩码优化 → 形态学操作
4. 边缘优化 → 羽化处理
5. 背景替换 → 合成新背景
```

#### 肤色检测范围 (HSV)

| 范围 | H范围 | S范围 | V范围 | 说明 |
|-----|-------|-------|-------|------|
| 范围1 | 0-20 | 20-255 | 70-255 | 偏黄肤色 |
| 范围2 | 170-180 | 20-255 | 70-255 | 偏红肤色 |

### 5. 质量评估 (Quality Assessment)

#### 评估维度

| 维度 | 权重 | 说明 |
|-----|------|------|
| 人脸检测 | 20% | 是否检测到人脸 |
| 正视度 | 15% | 是否正视摄像头 |
| 眼睛状态 | 15% | 眼睛是否睁开 |
| 嘴巴状态 | 10% | 嘴巴是否自然 |
| 眼镜检测 | 10% | 是否佩戴眼镜 |
| 光照评估 | 15% | 光照是否均匀 |
| 人脸大小 | 10% | 人脸是否适当大小 |

#### 评分计算

```python
# 综合评分
quality_score = (
    face_detection_score * 0.20 +
    frontal_score * 0.15 +
    eye_open_score * 0.15 +
    mouth_natural_score * 0.10 +
    no_glasses_score * 0.10 +
    lighting_score * 0.15 +

    level = '优秀'
elif quality_score >= 80:
    level = '良好'
elif quality_score >= 70:
    level = '一般'
else:
    level = '不合格'
```

## 常见问题

### Q1: 人脸检测失败怎么办？

**A**: 检查以下条件：
1. 光线是否充足
2. 人脸是否正视摄像头
3. 人脸是否过小或过大
4. 是否有遮挡物

### Q2: 人脸识别准确率不高怎么办？

**A**: 改进方法：
1. 确保采集的人脸清晰
2. 提高人脸特征的质量
3. 调整识别阈值
4. 使用多张人脸进行比对

### Q3: 人像分割边缘不清晰怎么办？

**A**: 优化方案：
1. 使用AI分割方法（rembg）
2. 调整边缘优化参数
3. 使用高质量的原始图像
4. 手动调整分割结果
