# 12 系统状态与报表说明

## 1. 采集系统四种状态

### 1.1 状态定义

系统采用三个数据库状态值（pending、processing、completed），显示时呈现四种状态（无记录、待采集、待处理、已完成）。

#### 1. 无记录（No Record）
- **定义**：用户信息存在于数据库，但没有采集记录
- **业务场景**：用户信息已导入系统，但还没有进行身份证读取
- **数据特征**：有用户记录（User表），无采集记录（CollectionRecord表）
- **显示**：无记录（灰色）
- **判断逻辑**：`if not records`

#### 2. 待采集（Pending）
- **定义**：已读取身份证，但还没有拍照
- **业务场景**：用户已将身份证放在读卡器上，系统读取了信息并创建了采集记录，但用户还没有拍照
- **数据特征**：有用户记录（User表），有采集记录，状态为 `pending`（CollectionRecord表）
- **显示**：待采集（浅黄色）
- **判断逻辑**：`records[-1].status == 'pending'`

#### 3. 待处理（Processing）
- **定义**：已拍照，但还没有进行图像处理
- **业务场景**：用户已完成拍照，系统保存了原始照片，但还没有进行背景替换、美颜等处理
- **数据特征**：有用户记录（User表），有采集记录，状态为 `processing`（CollectionRecord表），有原始照片（Photo表，photo_type='raw'）
- **显示**：待处理（浅橙色）
- **判断逻辑**：`records[-1].status == 'processing'`

#### 4. 已完成（Completed）
- **定义**：已完成拍照和图像处理
- **业务场景**：用户已完成整个采集流程，原始照片和处理后照片都已保存
- **数据特征**：有用户记录（User表），有采集记录，状态为 `completed`（CollectionRecord表），有原始照片（Photo表，photo_type='raw'），有处理后照片（Photo表，photo_type='processed'）
- **显示**：已完成（浅绿色）
- **判断逻辑**：`records[-1].status == 'completed'`

### 1.2 状态流转

```
[用户导入] → 无记录（没有采集记录）
    ↓
[读取身份证] → 待采集 (pending)
    ↓
[拍照] → 待处理 (processing)
    ↓
[图像处理] → 已完成 (completed)
```

### 1.3 数据库状态 vs 显示状态

| 显示状态 | 判断逻辑 | 数据库status | 说明 |
|---------|---------|-------------|------|
| 无记录 | `if not records` | 无记录 | 没有采集记录 |
| 待采集 | `status == 'pending'` | pending | 已读身份证，未拍照 |
| 待处理 | `status == 'processing'` | processing | 已拍照，未处理 |
| 已完成 | `status == 'completed'` | completed | 已拍照并处理 |

**关键点**：数据库中有三种状态值：`pending`、`processing`、`completed`

## 2. 采集进度趋势 - 三率对比

### 2.1 三率定义

采集进度趋势图显示三条趋势线，全面展示证件照采集流程的各个阶段进度：

#### 完成率（绿色线）
- **定义**：截止到当天，已完成整个采集流程的用户占比
- **计算公式**：完成率 = 截止到当天累计已完成数 / 用户总数 × 100%
- **业务含义**：用户已完成拍照 + 图像处理，状态为 `completed`

#### 处理率（橙色线）
- **定义**：截止到当天，至少已拍照的用户占比
- **计算公式**：处理率 = 截止到当天累计(已完成 + 待处理)数 / 用户总数 × 100%
- **业务含义**：用户已拍照，可能已处理或待处理，状态为 `completed` 或 `processing`

#### 采集率（蓝色线）
- **定义**：截止到当天，至少已开始采集流程的用户占比
- **计算公式**：采集率 = 截止到当天累计(已完成 + 待处理 + 待采集)数 / 用户总数 × 100%
- **业务含义**：用户已读取身份证，进入采集流程，状态为 `completed`、`processing` 或 `pending`

### 2.2 三率关系

**数学关系**：采集率 ≥ 处理率 ≥ 完成率

**三条线之间的间隙代表各阶段的"积压"**：
- 采集率 - 处理率 = 待采集率（已读身份证但还没拍照的用户占比）
- 处理率 - 完成率 = 待处理率（已拍照但还没处理的用户占比）
- 100% - 采集率 = 无记录率（还没开始采集流程的用户占比）

## 3. 导出报表功能

### 3.1 功能概述

优化后的导出报表功能可以生成一个包含8个数据表的Excel文件，提供全面的统计分析数据。

### 3.2 导出的数据表

#### Sheet 1：总体统计
包含整体概览信息：
- 采集任务名称
- 统计日期范围
- 报表生成时间
- 用户总数、照片总数
- 各状态人数（已完成、待处理、待采集、无记录）
- 完成率百分比
- 性别分布（男性人数、女性人数、性别比例）

#### Sheet 2：用户详细列表
每个用户的详细信息：
- 序号、姓名、身份证号、性别、民族、年龄、地区
- 采集状态、原始照片数、处理后照片数、采集日期

#### Sheet 3：状态分布
采集状态的统计分布：
- 已完成、待处理、待采集、无记录的人数和占比

#### Sheet 4：地区分布
按地区统计用户分布，按人数降序排列，显示占比百分比

#### Sheet 5：民族分布
按民族统计用户分布

#### Sheet 6：性别分布
按性别统计用户分布

#### Sheet 7：年龄分布
按年龄段统计用户分布

#### Sheet 8：每日统计
按采集日期统计每天的采集情况

### 3.3 导出流程

```
选择采集任务 → 选择日期范围 → 点击"导出Excel" 
→ 系统生成8个数据表 → 保存为Excel文件 → 提示导出成功
```

### 3.4 使用场景

- **管理者**：查看采集进度和统计数据
- **学校**：生成学生证件照统计报表
- **企业**：生成员工证件照统计报表
- **数据分析**：导出数据进行深度分析

## 4. 人脸检测流程（实际实现）

### 4.1 检测流程图

```
┌─────────────────────────────────────────────────────────────┐
│                    人脸检测流程（实际）                      │
└─────────────────────────────────────────────────────────────┘

输入图像 (BGR格式)
    ↓
尝试使用 fdlite (face-detection-tflite)
    ↓
检测成功？
    ├─ 是 → 获取检测结果（边界框、置信度）
    └─ 否 → 降级到 OpenCV 级联分类器
            ↓
            获取检测结果
    ↓
坐标转换
    ├─ 如果是归一化坐标(0-1) → 乘以图像宽高转换为像素坐标
    └─ 如果已是像素坐标 → 直接使用
    ↓
验证检测框合理性
    ├─ 宽度 > 20px 且 高度 > 20px → 保留
    └─ 否则 → 丢弃
    ↓
输出结果
    ├─ 边界框坐标 (x, y, width, height)
    ├─ 置信度评分 (0-1)
    └─ 检测框数量
```

### 4.2 修正说明

**当前论文描述**：
```
输入图像 → RGB转换 → fdlite检测 → 坐标转换 → 输出边界框和置信度
```

**实际项目流程**：
1. 添加"降级机制"：当fdlite不可用时自动降级到OpenCV级联分类器
2. 添加"检测框验证"：确保检测框的宽高大于20像素
3. 添加"坐标转换"的具体说明：从归一化坐标(0-1)转换为像素坐标

## 5. 人脸识别流程（实际实现）

### 5.1 识别流程图

```
身份证照片 → 人脸检测 → 人脸对齐 → 特征提取 → 特征向量1 (128维)
                                              ↓
                                          欧氏距离计算
                                              ↓
                                          相似度转换
                                          (1 - distance/2)
                                              ↓
                                          与阈值比较 (60%)
                                              ↓
                                          判断是否同一人
                                              ↑
拍摄照片 → 人脸检测 → 人脸对齐 → 特征提取 → 特征向量2 (128维)
```

### 5.2 修正说明

**当前论文描述**：
```
身份证照片 → 特征提取 → 特征向量1
                        ↓
                    距离计算 → 相似度 → 判断是否同一人
                        ↑
拍摄照片 → 特征提取 → 特征向量2
```

**实际项目流程**：
1. 添加"人脸检测"步骤：在特征提取前需要先检测人脸
2. 添加"人脸对齐"步骤：确保人脸处于标准姿态
3. 明确特征向量维度：128维
4. 明确相似度计算方式：1 - distance/2
5. 明确阈值设置：60%

## 6. 系统状态监控

### 6.1 实时状态显示

系统在主窗口状态栏显示实时统计信息：
- 总用户数
- 已完成数
- 待处理数
- 待采集数
- 无记录数
- 完成率百分比

### 6.2 状态更新时机

状态在以下操作后自动更新：
- 读取身份证（待采集数+1）
- 拍照（待处理数+1）
- 图像处理完成（已完成数+1）
- 删除用户（相应状态数-1）

### 6.3 状态查询

用户可以通过以下方式查询状态：
- 主窗口状态栏（实时显示）
- 数据管理模块（详细列表）
- 统计报表模块（图表展示）
- 导出Excel（数据导出）
