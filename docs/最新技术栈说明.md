# 最新技术栈说明（答辩版）

本文档基于代码实际实现与 `requirements.txt` 依赖清单整理，用于答辩时说明本课题系统所采用的功能模块、技术路线、关键模型与推理方式。

---

## 1. 系统总体架构与技术栈总览

系统是一个桌面端证件照“采集 + 图像处理 + 批量生成 + 数据管理”的一体化应用。整体分层如下：

- 表示层（UI）：`PyQt5`（窗体、控件、信号槽）
- 业务/控制层（算法编排）：`controllers/`（图像处理、背景替换、裁剪、人脸识别、批量处理、摄像头/读卡控制等）
- 算法与工具层：`utils/`（图像读写、质量评估、数据库访问、人脸特征管理等）
- 数据层：`SQLAlchemy`（SQLite 持久化）
- 图像与推理基础库：`OpenCV`、`numpy`、`Pillow`

### 主要依赖（来自 `requirements.txt`）

- 基础与图像处理：`numpy`、`opencv-python`、`Pillow`
- UI：`PyQt5`（在项目说明/代码中使用）
- 数据管理与报表：`SQLAlchemy`、`pandas`、`openpyxl`、`python-dateutil`
- 人脸检测（用于采集端质量检测与裁剪定位）：`face-detection-tflite`（fdlite）
- 背景移除/抠图（证件照背景替换）：`rembg`（内部使用 `u2net` 等分割模型）
- 人脸识别（身份核验/防重复采集）：`face-recognition`、`dlib`（dlib CNN/HOG 相关）
- 高保真商业管线（HiFi Pipeline）：`insightface`、`gfpgan`、`basicsr`、`facexlib`
- 可选推理加速框架：`onnxruntime`（代码中存在 ONNX 引擎类，但当前在主流程中不作为默认启用路径）
- 兼容性/底层：`tensorflow`（作为 tflite/推理相关的环境依赖出现）

## 2. 功能模块与对应技术实现

下面按“功能”说明模块职责、调用链路、以及使用的关键技术/模型。

### 2.1 主程序入口与界面导航

- 入口：`main.py`
  - 初始化目录与数据库：`config/database.py` + `config/config.py`
  - 启动 PyQt5：`views/main_window.py`
- 主窗口：`views/main_window.py`
  - `QTabWidget` 管理多页面：身份证读取、数据导入、拍照采集、图像处理、数据管理、统计报表

### 2.2 身份证读取模块（采集端信息获取）

- 控制器：`controllers/id_card_reader.py`
  - `IDCardReader` 负责读卡流程（当前代码提供测试模式，硬件 SDK 调用处预留）
  - 解析与校验：身份证号校验（校验位规则）、生日/性别解析
  - 生成身份证照片：`utils/id_card_photo_generator.py`（测试/模拟用）
- 数据存储：通过 `utils/database_helper.py` 写入 `models/` 对应表

说明：该模块在当前代码中以“硬件接口预留 + 测试模拟实现”方式呈现，因此答辩时可强调“采集硬件接入点已规划，当前以模拟数据保证系统端到端联调可跑通”。

### 2.3 摄像头拍照采集模块（质量控制 + 人脸检测）

- 摄像头控制：`controllers/camera.py`
  - `cv2.VideoCapture` 采集帧，使用线程循环读帧并回调
  - 提供摄像头参数读取（宽高/FPS/亮度/对比度等）
- 摄像头界面：`views/camera_view.py`
  - QThread 用于后台保存图片与后台相似度计算，避免阻塞 UI
  - 采集端人脸相关能力由：
    - `controllers/identity_verifier.py`（身份核验）
    - `controllers/duplicate_checker.py`（防重复）
    - `controllers/ai_processor.py`（人脸检测与质量评估）

核心思想：采集端更强调“实时性”，因此优先使用轻量人脸检测（`face-detection-tflite/fdlite` 或 OpenCV 级联降级）。

### 2.4 图像处理模块（裁剪 + 背景替换 + 美颜 + 参数调整）

图像处理入口在 `views/process_view.py`，通过 `controllers/image_processor.py` 完成算法编排。

#### 2.4.1 智能裁剪（按证件照规格定位）

- 控制器：`controllers/image_processor.py`（对外接口：`crop_to_spec` / `preview_crop`）
- 核心算法：`controllers/smart_cropper.py`（`SmartCropper.smart_crop`）
  - 规格参数来源：`config/config.py` 中的 `PHOTO_SPECS`
  - 人脸检测输入：使用 `controllers/ai_processor.py` 的 `detect_face` 结果作为定位依据
  - 裁剪策略：
    - 有人脸：基于人脸框估算“包含头发的完整头部高度”，再依据证件照比例计算上下边距与裁剪区域
    - 无人脸：回退中心裁剪（`center_based`）
  - 最终输出：裁剪后再 `resize` 到目标像素尺寸（如 590x826 等），并返回裁剪信息（用于调试/展示）

#### 2.4.2 背景替换（精确模式 refined + 高保真模式 HiFi）

背景替换在 `controllers/image_processor.py` 中提供两条路径：

- 精确背景替换：`controllers/background_replacer.py`
  - 类：`PreciseBackgroundReplacer`
  - 分割/抠图核心：`rembg`（在代码中显式以 `u2net` session 启用）
  - 关键流程：
    - 通过 `rembg.remove(..., alpha_matting=True, alpha_matting_*阈值)` 获得 RGBA 输出
    - 从 alpha 通道提取 mask，并进行阈值/二值化
    - 使用多层边缘羽化、头发丝特殊混合、衣服边缘抗锯齿等传统图像处理策略提升合成自然度
  - 结果：得到指定背景色（`config/config.py` 的 `BACKGROUND_COLORS`）的合成图像

- 高保真商业管线：`controllers/hifi_pipeline.py`（`HiFiPipeline`）
  - 模型链路（代码开头说明与实现一致）：
    1. `InsightFace`：人脸检测与身份特征提取（用于定位/锁定特征，代码中使用 `FaceAnalysis(name='buffalo_l')`）
    2. 语义分割（当前以 `rembg` 作为 MODNet 替代）：生成原始 Alpha Mask（并单独做平滑）
    3. 可选细节增强：优先 `GFPGAN`（代码中通过 `gfpgan.GFPGANer`，权重路径在实现处给出）；`CodeFormer` 在代码中标记为可选且未真正集成
    4. Alpha mask 平滑：Canny 检测边缘 + 高斯模糊 + adaptiveThreshold，避免锯齿边缘
    5. 最终合成：使用平滑后的 mask 做背景合成
  - 推理设备：`insightface` 以 `CPUExecutionProvider/CPUProvider` 配置（代码中显式指定）

#### 2.4.3 美颜与瑕疵处理（选择性美颜）

- 选择性美颜：`controllers/advanced_beautify.py`（`AdvancedBeautify.selective_beautify`）
  - 使用 `ai_processor/detect_face` 给到人脸框
  - 在人脸区域内进行传统 CV 操作：
    - 磨皮：双边滤波 + 高斯模糊混合（保留一定细节）
    - 祛痘：形态学（tophat/blackhat）+ 阈值 + mask 限制 + Inpainting 修复
    - 眼部/唇部/牙齿等区域增强：根据人脸框估算区域并进行局部处理（实现中以 mask/区域几何为主）

#### 2.4.4 实时参数调整（亮度/对比度/饱和度/锐化等）

- 模块：`controllers/realtime_adjuster.py`
  - 使用 `numpy` + OpenCV 做像素级变换（亮度、对比度、gamma、色彩相关、清晰度等）
  - 提供预设参数（如“标准证件照”“护照照片”等）
  - GUI 与预览联动通过 `update_callback` 完成

### 2.5 对比显示交互（UI 交互逻辑）

- `views/process_view.py`
  - “对比”按钮实现为“长按显示原图、松开恢复处理后图”的交互逻辑
  - 该交互属于展示逻辑，不改变底层处理链路

### 2.6 裁剪对话框（规格选择）

- `views/crop_dialog.py`
  - 裁剪规格下拉来自配置 `config/config.py::PHOTO_SPECS`（保证规格完整性）
  - 用户可拖拽裁剪框，并最终输出裁剪结果再进入处理链路

### 2.7 多规格与批量生成（统一批量处理）

- 批量处理器：`controllers/batch_processor.py`
  - 以队列方式处理图片（输入文件列表 → 逐张处理 → 输出文件）
  - 使用 Python `threading.Thread` 在后台跑处理循环，避免 UI 卡顿
  - 根据批量参数 `batch_params` 决定处理模式：
    - 单规格：`crop_spec` + 单颜色背景
    - 多规格模式：`multi_specs` × `multi_colors` 组合生成
  - 图像处理由 `controllers/image_processor.py` 复用同一套算法入口

- 批量 UI：
  - `views/batch_processing_dialog.py`（批量处理对话框）
  - `views/unified_batch_dialog.py`（融合后的“多规格批量生成”，支持单张/多张输入）
  - 并通过 `QThread` + 信号将进度更新回界面

### 2.8 身份核验与防重复采集（特征编码 + 阈值比对）

- 统一特征管理：`utils/face_recognition_manager.py`
  - 运行模式选择：
    - 若安装 `face_recognition`：走 dlib CNN 体系（HOG 方案用于定位 + face_encodings 提取向量）
    - 否则回退 OpenCV 简化特征（用于演示/保底）
  - 输出：128 维人脸特征向量（face_recognition 路径）或简化编码（OpenCV 路径）
  - 比对策略：向量距离（欧几里得）与 tolerance 阈值判定

- 防重复：`controllers/duplicate_checker.py`
  - 从数据库读取用户已存储的人脸编码
  - 新采集图像编码后计算相似度，达到阈值即判定为重复

- 身份核验：`controllers/identity_verifier.py`
  - 核验流程：
    1. 从数据库找到用户的身份证照片（id_photo_data）
    2. 提取身份证照片与当前采集照片的人脸特征向量
    3. 根据阈值计算相似度，输出 `verified` 与相似度百分比

#### 可选人脸引擎（代码中存在，但当前主流程以 FaceRecognitionManager 为中心）

- `utils/face_recognition_engine.py`（dlib 引擎封装，可选）
- `utils/face_recognition_engine_insightface.py`
  - 使用 `insightface.app.FaceAnalysis(name='buffalo_l')` 提取 embedding
  - 使用余弦相似度映射到 0-1，并与阈值比较
- `utils/face_recognition_engine_onnx.py`
  - 类似“ONNX FaceNet”的接口封装，但 `_load_model()` 中体现为加载占位/演示结构（需要模型文件时才可真正启用）
- `utils/face_recognition_engine_cv.py`
  - OpenCV 级联 + HOG/直方图等传统特征编码

答辩表达建议：强调“主流程已落地（FaceRecognitionManager + dlib/ OpenCV 退化），同时预留了 insightface/onnx 的可扩展引擎类，方便后续升级精度与速度”。

### 2.9 数据管理、统计报表与导出

- 数据持久化：`config/database.py` + `utils/database_helper.py` + `models/*`
  - SQLAlchemy ORM，底层数据库为 `SQLite`（`config/config.py::DATABASE_URL`）
  - 主要模型：`User`、`Photo`、`Collection`、`CollectionRecord`
  - 人脸特征编码存储：将向量转字节并持久化（见 duplicate/identity 的读取方式）
- 统计报表：`views/report_view.py`
  - 使用 `pandas` 聚合统计
  - 通过 `openpyxl` 导出 Excel（UI 中提供“导出报表”功能）
- 数据导入：`views/import_view.py`
  - 支持手动录入与 CSV/Excel 批量导入（代码内体现 `csv` 等逻辑）

### 2.10 语音提示（交互增强）

- `utils/voice_helper.py`
  - 使用 `pyttsx3` 离线语音合成进行提示音播放
  - 通过独立线程异步播放并设置最小播放间隔避免重叠

## 3. 深度学习/大模型（或深度推理）清单与角色

本系统使用的“深度学习模型/深度推理组件”主要来自以下库与内部模型：

1. `face-detection-tflite` / `fdlite`
   - 角色：采集端人脸检测（用于质量判断、裁剪定位输入）
   - 形态：tflite 轻量检测器（代码中 `FaceDetection()`）

2. `rembg`（U2-Net / isnet-general-use 等）
   - 角色：背景移除/人像抠图（生成 mask/alpha 通道）
   - 在代码中显式以：
     - `PreciseBackgroundReplacer`：`rembg.new_session('u2net')` 并开启 `alpha_matting=True`
     - `HiFiPipeline`：作为 MODNet 替代生成原始 Alpha mask，并自行进行平滑与合成

3. `insightface`（`FaceAnalysis(name='buffalo_l')`，embedding 提取）
   - 角色：HiFiPipeline 的人脸检测/身份特征提取（身份锁定与定位辅助）
   - 在人脸引擎封装类中也作为可选路径存在（余弦相似度比对）

4. `gfpgan`（GFPGAN 细节增强，可选）
   - 角色：HiFiPipeline 的可选细节增强（人像主体 ROI 内增强）
   - 代码中给出 GFPGAN 权重路径示例，并在运行时按可用性降级/跳过

5. `CodeFormer`
   - 角色：计划用于“保真细节增强”
   - 当前实现：在 `HiFiPipeline` 中明确标记为“可选但未真正集成/待集成”（答辩时可作为后续工作点）

6. `dlib` / `face_recognition`
   - 角色：身份核验、防重复采集（人脸特征向量编码 + 阈值比对）
   - 形态：通过 `face_recognition.face_locations(..., model='hog')` 定位，人脸编码通过 `face_recognition.face_encodings` 获取 embedding

7. `onnxruntime`（可选）
   - 角色：预留 ONNX 人脸引擎接口（`FaceRecognitionEngineONNX`）
   - 当前状态：代码体现为接口封装/占位逻辑，实际需配套模型文件才可真正作为生产推理引擎启用

## 4. 推理设备与性能/稳定性策略

- HiFiPipeline：
  - `insightface` 明确使用 CPU providers（代码中配置为 `CPUExecutionProvider/CPUProvider`）
  - `rembg` 通过 `session` 加载（`u2net` / `isnet-general-use`），并在关键路径中复用 session
  - Alpha mask 平滑与合成全部为 CPU 图像处理（OpenCV）

- AIProcessor（采集端检测）：
  - 优先使用轻量 tflite 检测（`fdlite`）
  - 若不可用则使用 OpenCV Haar cascade 降级
  - 检测稳定性：维护检测历史窗口，只有连续多次检测到才认为稳定（降低抖动对后续裁剪/质量判断的影响）

- UI 卡顿控制：
  - 相似度计算与保存操作使用 `QThread` / `threading.Thread`
  - 批量处理以后台线程串行处理队列，并通过回调/信号反馈进度

## 5. 答辩时可重点强调的“工程化亮点”

- 统一算法入口：图像处理链路由 `ImageProcessor` 统一编排，批量与单次处理复用同一套裁剪/背景替换/美颜逻辑，降低维护成本。
- 模型可用性与优雅降级：
  - 采集端人脸检测：tflite 不可用就退化 OpenCV cascade
  - 人脸识别：未安装 dlib 系统就退化 OpenCV 简化编码
  - HiFiPipeline：MODNet 缺失时用 `rembg` 作为替代；GFPGAN 不可用则跳过细节增强
  - CodeFormer 在当前实现中明确为后续集成项（答辩可作为展望）
- 输出自然度优化：在背景替换与 HiFiPipeline 中不仅做 mask 合成，还引入多层边缘羽化、头发丝特殊混合、Alpha mask 平滑等工程策略来提升视觉效果。

## 6. 主要代码入口（用于现场讲解的“从 UI 到模型”的路径）

1. 程序启动：`main.py` -> `views/main_window.py`
2. 拍照采集：`views/camera_view.py` -> `controllers/camera.py`（取帧）+ `controllers/identity_verifier.py` / `controllers/duplicate_checker.py`（核验/去重）
3. 单张图像处理：`views/process_view.py` -> `controllers/image_processor.py`
   - 智能裁剪：`SmartCropper`（`controllers/smart_cropper.py`）
   - 背景替换：`PreciseBackgroundReplacer` / `HiFiPipeline`
   - 美颜：`AdvancedBeautify`
   - 实时参数：`RealTimeAdjuster`
4. 批量/多规格生成：`views/batch_processing_dialog.py` / `views/unified_batch_dialog.py`
   -> `controllers/batch_processor.py`（后台队列处理）-> 复用 `ImageProcessor`
