# 03 数据库设计

## 3.1 数据库概述

**数据库类型**: SQLite  
**ORM框架**: SQLAlchemy  
**文件位置**: `data/database/id_photo.db`

**设计原则**:
- 规范化设计，减少冗余
- 合理使用外键，保证完整性
- 适当冗余，提高查询效率
- 索引优化，提升性能

## 3.3 数据表设计

### 3.3.1 Collection (采集任务表)

**用途**: 管理不同的采集任务，实现数据隔离

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Integer | PK, AI | 主键，自增 |
| name | String(100) | NOT NULL | 任务名称 |
| organization | String(200) | NOT NULL | 组织机构名称 |
| description | Text | NULL | 任务描述 |
| start_date | Date | NOT NULL | 开始日期 |
| end_date | Date | NULL | 结束日期 |
| status | String(20) | NOT NULL | 状态: active/completed/archived |
| created_at | DateTime | NOT NULL | 创建时间 |
| updated_at | DateTime | NOT NULL | 更新时间 |

**索引**:
- PRIMARY KEY (id)
- INDEX idx_status (status)
- INDEX idx_created_at (created_at)

**示例数据**:
```sql
INSERT INTO collection VALUES (
    1, '2024年新生入学', '某某大学', 
    '2024级新生证件照采集', '2024-09-01', '2024-09-30',
    'active', '2024-08-15 10:00:00', '2024-08-15 10:00:00'
);
```

### 3.3.2 User (用户信息表)

**用途**: 存储用户基本信息

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Integer | PK, AI | 主键，自增 |
| collection_id | Integer | FK, NOT NULL | 所属采集任务ID |
| name | String(50) | NOT NULL | 姓名 |
| id_number | String(18) | NOT NULL | 身份证号 |
| gender | String(10) | NULL | 性别: 男/女 |
| nation | String(20) | NULL | 民族 |
| birthday | Date | NULL | 出生日期 |
| address | Text | NULL | 地址 |
| created_at | DateTime | NOT NULL | 创建时间 |
| updated_at | DateTime | NOT NULL | 更新时间 |

**约束**:
- FOREIGN KEY (collection_id) REFERENCES collection(id) ON DELETE CASCADE
- UNIQUE (collection_id, id_number) - 同一任务内身份证号唯一

**索引**:
- PRIMARY KEY (id)
- INDEX idx_collection_id (collection_id)
- INDEX idx_id_number (id_number)
- INDEX idx_name (name)

**字段说明**:
- `id_number`: 18位身份证号，用于唯一标识和地区推断
- `birthday`: 从身份证号提取或手动输入
- `address`: 优先使用身份证地址，为空时从身份证号推断

**示例数据**:
```sql
INSERT INTO user VALUES (
    1, 1, '张三', '110101199001011234', 
    '男', '汉族', '1990-01-01', '北京市东城区XX街道',
    '2024-09-01 09:00:00', '2024-09-01 09:00:00'
);
```

### 3.3.3 Photo (照片记录表)

**用途**: 记录照片文件信息

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Integer | PK, AI | 主键，自增 |
| user_id | Integer | FK, NOT NULL | 所属用户ID |
| photo_type | String(20) | NOT NULL | 类型: raw/processed |
| file_path | String(500) | NOT NULL | 文件路径 |
| file_size | Integer | NULL | 文件大小(字节) |
| photo_spec | String(50) | NULL | 照片规格(如: 一寸/二寸) |
| background_color | String(50) | NULL | 背景颜色 |
| created_at | DateTime | NOT NULL | 创建时间 |

**约束**:
- FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE

**索引**:
- PRIMARY KEY (id)
- INDEX idx_user_id (user_id)
- INDEX idx_photo_type (photo_type)

**字段说明**:
- `photo_type`: 
  - `raw`: 原始照片（摄像头拍摄）
  - `processed`: 处理后照片（美颜/背景/裁剪后）
- `photo_spec`: 如"一寸"、"二寸"、"35x45mm"等
- `background_color`: 如"white"、"blue"、"red"等

**示例数据**:
```sql
-- 原始照片
INSERT INTO photo VALUES (
    1, 1, 'raw', 'data/photos/user_1_raw_20240901_090530.jpg',
    245678, NULL, NULL, '2024-09-01 09:05:30'
);

-- 处理后照片
INSERT INTO photo VALUES (
    2, 1, 'processed', 'data/photos/user_1_processed_20240901_090545_一寸_white.jpg',
    189234, '一寸', 'white', '2024-09-01 09:05:45'
);
```

### 3.3.4 CollectionRecord (采集记录表)

**用途**: 记录用户采集状态和历史

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Integer | PK, AI | 主键，自增 |
| collection_id | Integer | FK, NULL | 所属采集任务ID |
| user_id | Integer | FK, NOT NULL | 用户ID |
| operator | String(50) | NULL | 操作员 |
| status | String(20) | NOT NULL | 状态 |
| collection_date | Date | NOT NULL | 采集日期 |
| notes | Text | NULL | 备注 |
| created_at | DateTime | NOT NULL | 创建时间 |

**约束**:
- FOREIGN KEY (collection_id) REFERENCES collection(id) ON DELETE SET NULL
- FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE

**索引**:
- PRIMARY KEY (id)
- INDEX idx_user_id (user_id)
- INDEX idx_collection_date (collection_date)
- INDEX idx_status (status)

**字段说明**:
- `status`: 采集状态
  - `pending`: 待采集
  - `completed`: 已完成
  - `failed`: 失败
- `collection_date`: 实际采集日期
- `notes`: 失败原因或其他备注

**示例数据**:
```sql
INSERT INTO collection_record VALUES (
    1, 1, 1, '操作员A', 'completed', 
    '2024-09-01', '采集成功', '2024-09-01 09:05:50'
);
```

## 3.4 数据完整性

### 3.4.1 实体完整性
- 所有表都有主键
- 主键自增，保证唯一性

### 3.4.2 参照完整性
- 外键约束保证关联数据一致性
- 级联删除：删除采集任务/用户时，自动删除关联数据
- 级联更新：更新主键时，自动更新外键

### 3.4.3 用户定义完整性
- 身份证号格式校验（18位数字）
- 状态字段枚举值限制
- 日期合理性校验

## 3.5 查询优化

### 3.5.1 常用查询

**查询用户及其照片**:
```sql
SELECT u.*, p.photo_type, p.file_path
FROM user u
LEFT JOIN photo p ON u.id = p.user_id
WHERE u.collection_id = ?
ORDER BY u.created_at DESC;
```

**统计采集状态**:
```sql
SELECT status, COUNT(*) as count
FROM collection_record
WHERE collection_id = ?
  AND collection_date BETWEEN ? AND ?
GROUP BY status;
```

**查询待采集用户**:
```sql
SELECT u.*
FROM user u
LEFT JOIN collection_record r ON u.id = r.user_id
WHERE u.collection_id = ?
  AND (r.status IS NULL OR r.status = 'pending');
```

### 3.5.2 索引策略

**已创建索引**:
- 主键索引（自动）
- 外键索引（collection_id, user_id）
- 查询字段索引（id_number, name, status, collection_date）

**索引选择原则**:
- 经常作为WHERE条件的字段
- 经常作为JOIN条件的字段
- 经常作为ORDER BY的字段
- 区分度高的字段
