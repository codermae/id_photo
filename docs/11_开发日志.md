### v1.0.0 (2025-09-15) - 项目启动
**目标**: 搭建基础框架，实现核心功能

**完成内容**:
- 项目结构设计（MVC架构）

- 身份证读取模块（测试模式）

**技术决策**:
1. 选择PyQt5作为UI框架

   - 备选方案：Tkinter（功能较弱）、wxPython（学习曲线陡）

2. 选择SQLite作为数据库
   - 理由：轻量级、无需安装、适合单机应用
   - 备选方案：MySQL（需要服务器）、PostgreSQL（过于复杂）

3. 选择OpenCV作为图像处理库
   - 理由：功能全面、性能优秀、社区活跃
   - 备选方案：PIL（功能较弱）、scikit-image（速度较慢）

### v1.1.0 (2025-10-20) - AI功能集成
**目标**: 集成AI算法，提升处理质量

**完成内容**:
- 集成face-detection-tflite人脸检测
- 集成face_recognition身份核验
- 集成U2-Net背景分割
- 集成GFPGAN人脸美颜

**遇到的问题**:
1. **GFPGAN模型加载慢**
   - 问题：首次加载耗时5-10秒
   - 解决：在程序启动时预加载模型
   - 代码位置：`controllers/ai_processor.py` 的 `__init__` 方法

2. **U2-Net内存占用大**
   - 问题：处理高分辨率图像时内存占用超过2GB
   - 解决：先缩放到1024x1024，处理后再放大
   - 代码位置：`controllers/background_replacer.py` 的 `segment_portrait` 方法

3. **face-detection-tflite在某些电脑上无法使用GPU**
   - 问题：部分电脑没有CUDA支持
   - 解决：自动检测，无GPU时使用CPU模式
   - 代码位置：`controllers/ai_processor.py` 的模型初始化部分

**性能优化**:
- 人脸检测：从200ms优化到50ms（使用BlazeFace模型）
- 背景分割：从3秒优化到1.5秒（图像预缩放）
- 美颜处理：从2秒优化到1秒（批量处理优化）

**技术难点**:
1. **多线程UI更新**
   - 问题：后台线程无法直接更新UI
   - 解决：使用QThread和pyqtSignal机制
   - 示例代码：
   ```python
   class ProcessThread(QThread):
       progress = pyqtSignal(int, int, str)
       
       def run(self):
           for i, image in enumerate(images):
               process(image)
               self.progress.emit(i+1, total, image_path)
   ```

2. **内存管理**
   - 问题：批量处理大量图像导致内存溢出
   - 解决：处理完一张立即释放内存
   - 代码：
   ```python
   processed = process_image(image)
   save_image(processed)
   del image, processed
   gc.collect()
   ```

   - 解决：使用线程锁保护数据库操作
   ```python
   db_lock = threading.Lock()
   with db_lock:
       db.execute(query)
   ```

3. **摄像头释放问题**
   - 问题：切换摄像头时旧摄像头未释放
   - 解决：切换前先调用 `camera.release()`

## 技术难点总结

### 1. 身份核验算法
**难点**: 如何准确判断摄像头中的人和身份证照片是同一人

**解决方案**:
- 使用face_recognition库提取128维人脸特征向量
- 计算欧氏距离判断相似度
- 设置阈值60%（经过测试调优）

**关键代码**:
```python
id_encoding = face_recognition.face_encodings(id_photo)[0]
camera_encoding = face_recognition.face_encodings(camera_frame)[0]
distance = np.linalg.norm(id_encoding - camera_encoding)
similarity = 1.0 - min(distance / 2.0, 1.0)
```

### 2. 背景精细抠图
**难点**: 如何实现头发丝级别的精细抠图

**解决方案**:
- 使用U2-Net深度学习模型
- 模型专门训练用于显著性目标检测
- 输出高质量的人像掩码

**优化措施**:
- 图像预缩放到1024x1024（减少计算量）
- 使用GPU加速（如果可用）
- 提供快速模式（GrabCut算法）供选择

### 3. 实时人脸检测
**难点**: 如何在30fps下实时检测人脸

**解决方案**:
- 使用face-detection-tflite的BlazeFace模型
- 轻量级模型，CPU上可达30fps
- 检测结果包含边界框和关键点

**性能对比**:
- dlib: 5fps (CPU)
- OpenCV Haar: 15fps (CPU)
- face-detection-tflite: 30fps (CPU)

### 4. 多线程UI更新
**难点**: 后台线程如何安全更新UI

**解决方案**:
- 使用PyQt5的QThread和信号槽机制
- 后台线程通过信号发送数据
- 主线程接收信号更新UI

**示例**:
```python
class WorkerThread(QThread):
    result_ready = pyqtSignal(object)
    
    def run(self):
        result = do_heavy_work()
        self.result_ready.emit(result)

# 主线程
thread = WorkerThread()
thread.result_ready.connect(self.update_ui)
thread.start()
```

### 5. 图表交互实现
**难点**: 如何在PyQt中实现交互式图表

**解决方案**:
- 使用ECharts生成HTML图表
- 使用QWebEngineView渲染HTML
- 通过JavaScript实现交互功能

**技术栈**:
- Python生成图表配置
- ECharts渲染图表
