# 核心模块详解

## 1. 身份证读取模块

### 1.1 模块概述

身份证读取模块负责通过身份证读卡器读取用户的身份信息和照片，是整个系统的数据入口。该模块支持0501SAM模块读卡器，并提供测试模式用于开发调试。

### 1.2 核心组件

#### 1.2.1 IDCardReader 控制器
**文件路径**: controllers/id_card_reader.py

**主要功能**:
- 读卡器连接管理
- 身份证信息读取
- 身份证号码校验
- 测试模式支持

**关键方法**:

`python
def connect(self) -> bool
`
连接身份证读卡器硬件，初始化SDK。返回连接是否成功。

`python
def start_reading(self, callback=None) -> bool
`
启动连续读卡模式，在后台线程中循环检测身份证。当检测到新卡片时，调用回调函数传递卡片数据。

`python
def _read_loop(self)
`
读卡循环线程，每100ms检测一次卡片状态。当检测到新身份证时触发回调，当身份证移除时重置状态。

`python
def validate_id_number(self, id_number) -> bool
`
验证身份证号码的合法性，使用加权校验码算法验证最后一位校验码。

**数据结构**:
`python
card_data = {
    'name': str,              # 姓名
    'id_number': str,         # 18位身份证号
    'gender': str,            # 性别（男/女）
    'nation': str,            # 民族
    'birthday': str,          # 出生日期 YYYY-MM-DD
    'address': str,           # 住址
    'issue_authority': str,   # 签发机关
    'valid_period': str,      # 有效期限
    'id_photo': bytes,        # 身份证照片二进制数据
    'read_time': str          # 读取时间 ISO格式
}
`

#### 1.2.2 IDCardView 视图
**文件路径**: iews/id_card_view.py

**界面布局**:
- 采集任务选择区：显示当前任务，选择目标采集任务

- 读卡器状态区：显示连接状态（已连接/未连接）

**关键功能**:

1. **读卡器连接**
`python
def connect_reader(self)
`
点击"启动读卡"按钮时调用，连接读卡器硬件并启动连续读卡模式。

2. **卡片数据处理**
`python
def on_card_read(self, card_data)
`
当读取到身份证时的回调函数，执行以下流程：
- 验证身份证号码合法性
- 检查用户是否已存在数据库

3. **测试模式**

### 1.3 业务流程

#### 1.3.1 正常读卡流程
`
用户放置身份证 → 读卡器检测到卡片 → 读取芯片数据 → 验证身份证号 
→ 查询数据库是否存在 → 显示用户信息 → 发出用户选中信号
`

**详细步骤**:
1. 用户点击"启动读卡"按钮
2. 系统调用 connect_reader() 连接读卡器
3. 启动后台线程 _read_loop() 循环检测
4. 用户将身份证放在读卡器上
5. 检测到卡片，调用 _read_card() 读取数据
6. 触发 on_card_read() 回调函数
7. 验证身份证号码（加权校验码算法）
8. 查询数据库：db.get_user_by_id_number()
9. 如果是新用户且开启自动添加：
   - 调用 db.add_user() 添加用户
   - 调用 db.add_user_to_collection() 加入采集任务
10. 更新界面显示信息和照片
11. 发出 user_selected 信号（携带 user_id, name, photo, collection_id）
12. 其他模块（摄像头、图像处理）接收信号并更新状态

#### 1.3.2 身份证移除流程
`
用户拿走身份证 → 读卡器检测不到卡片 → 清空显示信息 → 发出用户清空信号
`

### 1.4 技术要点

#### 1.4.1 身份证号码校验算法.00
使用 GB 11643-1999 标准的加权校验码算法：

`python
weights = [7, 9, 10, 5, 8, 4, 2, 1, 6, 3, 7, 9, 10, 5, 8, 4, 2]
check_codes = '10X98765432'

# 计算加权和
total = sum(int(id_number[i]) * weights[i] for i in range(17))
# 取模得到校验码
check_code = check_codes[total % 11]
# 验证最后一位
return id_number[17] == check_code
`

#### 1.4.2 多线程读卡
使用 QThread 在后台线程中循环读卡，避免阻塞UI：
- 主线程：UI交互和显示
- 读卡线程：每100ms检测一次卡片状态
- 通过 pyqtSignal 在线程间传递数据

#### 1.4.3 测试模式
为方便开发调试，实现了测试模式：
- 使用 IDCardPhotoSimulator 生成模拟身份证照片
- 预设多组测试数据（不同性别、民族、地区）
- 点击"测试读卡"按钮模拟读卡操作

## 2. 摄像头拍照模块

### 2.1 模块概述

摄像头拍照模块负责实时预览摄像头画面、拍摄证件照、进行人脸检测和身份核验。该模块集成了AI算法，提供实时的人脸检测、质量评估和身份验证功能。

### 2.2 核心组件

#### 2.2.1 CameraController 控制器
**文件路径**: controllers/camera.py

**主要功能**:
- 摄像头设备管理
- 实时视频流获取
- 帧率控制
- 多摄像头支持

**关键方法**:

`python
def list_cameras(self) -> List[Dict]

枚举系统中所有可用的摄像头设备，返回设备列表（ID、名称、分辨率）。

`python
def open_camera(self, camera_id: int) -> bool
`
打开指定ID的摄像头，设置分辨率和帧率。

`python
def read_frame(self) -> np.ndarray
`
读取一帧图像，返回 BGR 格式的 numpy 数组。

#### 2.2.2 CameraView 视图
**文件路径**: iews/camera_view.py

**界面布局**:
- 摄像头选择区：选择摄像头设备、分辨率
- 实时预览区：显示摄像头画面（640x480）
- 人脸检测区：显示检测到的人脸数量、位置框
- 身份核验区：显示与身份证照片的相似度（0-100%）
- 质量评估区：显示照片质量评分和各项指标
- 拍照控制区：拍照按钮、倒计时设置、语音提示开关
- 照片预览区：显示已拍摄的照片缩略图

**关键功能**:

1. **实时预览**
`python
def update_frame(self)
`
定时器每33ms调用一次（30fps），执行以下流程：
- 从摄像头读取一帧图像
- 进行人脸检测（face-detection-tflite）
- 在画面上绘制人脸框和关键点
- 如果选中了用户，计算与身份证照片的相似度
- 更新预览画面

2. **人脸检测**
`python
def detect_face(self, frame)
`
使用 face-detection-tflite (BlazeFace) 检测人脸：
- 检测人脸数量（要求只有1个人脸）
- 提取人脸边界框坐标
- 在画面上绘制检测结果

3. **身份核验**
`python
def calculate_similarity(self, id_card_photo, current_frame)
`
在后台线程中计算相似度，避免阻塞UI：
- 使用 face_recognition 库提取人脸特征向量（128维）
- 计算欧氏距离
- 转换为相似度百分比：similarity = 1.0 - min(distance / 2.0, 1.0)
- 相似度 > 60% 认为是同一人

4. **拍照功能**
`python
def take_photo(self)
`
拍照流程：
- 检查是否选中用户
- 检查人脸检测结果（必须有且仅有1个人脸）
- 检查身份核验结果（相似度 > 60%）
- 保存原始照片到 data/raw_photos/
- 提取并保存人脸特征（用于后续去重）
- 添加照片记录到数据库
- 更新采集任务状态为"已采集"
- 语音提示"拍照成功"

5. **质量评估**
`python
def evaluate_photo_quality(self, frame)
`
评估照片质量，返回0-100分：
- 清晰度（30分）：拉普拉斯算子计算边缘清晰度
- 亮度（25分）：检查曝光是否适中
- 人脸大小（25分）：人脸占画面比例是否合适
- 人脸角度（20分）：是否正面、无遮挡

### 2.3 业务流程

#### 2.3.1 拍照完整流程
`
选择用户 → 打开摄像头 → 实时预览 → 人脸检测 → 身份核验 
→ 质量评估 → 点击拍照 → 保存照片 → 更新数据库 → 语音提示
`

**详细步骤**:
1. 用户在身份证读取模块选中用户
2. 系统接收 user_selected 信号
3. 更新界面显示用户信息和身份证照片
4. 用户选择摄像头并打开
5. 启动定时器，每33ms更新一帧
6. 每帧执行：
   - 人脸检测（face-detection-tflite）
   - 绘制人脸框和关键点
   - 如果有用户选中，启动后台线程计算相似度
   - 更新相似度显示（绿色>60%，黄色40-60%，红色<40%）
   - 评估照片质量并显示评分
7. 用户调整位置和角度，确保：
   - 只有1个人脸
   - 相似度 > 60%
   - 质量评分 > 70分
8. 点击"拍照"按钮
9. 系统验证条件是否满足
10. 启动后台线程保存照片：
    - 保存原始照片文件
    - 提取人脸特征向量
    - 添加照片记录到数据库

11. 语音提示"拍照成功"
12. 在照片预览区显示缩略图

### 2.4 技术要点

#### 2.4.1 人脸检测算法
使用 face-detection-tflite (BlazeFace) 模型：
- 模型：BlazeFace（轻量级，适合实时检测）
- 输入：RGB图像
- 输出：人脸边界框、置信度
- 性能：CPU上可达30fps

`python
# 使用face-detection-tflite进行人脸检测
from face_detection_tflite import FaceDetector

face_detector = FaceDetector(
    model_path='models/face_detection.tflite',
    min_confidence=0.5
)
`

#### 2.4.2 身份核验算法
使用 face_recognition 库（基于 dlib）：
- 人脸特征提取：128维特征向量
- 距离计算：欧氏距离
- 相似度转换：similarity = 1.0 - min(distance / 2.0, 1.0)
- 阈值设置：60%（可调整）

**为什么使用后台线程**:
特征提取和距离计算耗时约50-100ms，如果在主线程执行会导致预览卡顿。使用 QThread 在后台计算，计算完成后通过信号更新UI。

#### 2.4.3 照片质量评估
**清晰度评估**（30分）:
`python
# 使用拉普拉斯算子计算边缘强度
gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
laplacian_var = cv2.Laplacian(gray, cv2.CV_64F).var()
# 方差越大，图像越清晰
clarity_score = min(laplacian_var / 500 * 30, 30)
`

**亮度评估**（25分）:
`python
# 计算平均亮度
mean_brightness = np.mean(gray)
# 理想亮度：120-140
if 120 <= mean_brightness <= 140:
    brightness_score = 25
else:
    brightness_score = max(0, 25 - abs(mean_brightness - 130) / 2)
`

**人脸大小评估**（25分）:
`python
# 人脸占画面比例
face_ratio = face_area / frame_area
# 理想比例：15-30%
if 0.15 <= face_ratio <= 0.30:
    size_score = 25
else:
    size_score = max(0, 25 - abs(face_ratio - 0.225) * 100)
`

**人脸角度评估**（20分）:
`python
# 通过关键点计算偏航角、俯仰角、翻滚角
# 理想角度：正面（各角度 < 15度）
angle_score = 20 - (abs(yaw) + abs(pitch) + abs(roll)) / 3
`

使用 pyttsx3 库进行语音播报：
- 拍照成功："拍照成功"
- 检测到多个人脸："请保持只有一个人"
- 相似度不足："请确认是本人"
- 质量不合格："请调整位置和光线"

## 3. 图像处理模块

### 3.1 模块概述

图像处理模块是系统的核心，负责对拍摄的原始照片进行裁剪、换背景、美颜、调色等处理，生成符合标准的证件照。该模块集成了多种AI算法，提供高质量的图像处理能力。

### 3.2 核心组件

#### 3.2.1 ImageProcessor 控制器
**文件路径**: controllers/image_processor.py

**主要功能**:
- 图像裁剪（智能裁剪、手动裁剪）
- 背景替换（纯色、渐变、自定义）
- 图像增强（亮度、对比度、饱和度、锐化）
- 尺寸调整（一寸、二寸、小一寸等）
- 格式转换（JPG、PNG）
- DPI设置

**关键方法**:

`python
def crop_photo(self, image, spec_name, manual_rect=None)
`
裁剪照片为指定规格：
- spec_name: 规格名称（一寸、二寸等）
- manual_rect: 手动裁剪矩形（可选）
- 返回：裁剪后的图像

**裁剪逻辑**:
1. 如果提供 manual_rect，直接按矩形裁剪
2. 否则使用智能裁剪：
   - 检测人脸位置
   - 根据规格计算裁剪区域（人脸居中，头顶留白）
   - 确保裁剪区域在图像范围内

`python
def replace_background(self, image, color, mode='refined')
`
替换背景颜色：
- color: 目标颜色（RGB元组或颜色名称）
- mode: 抠图模式（'refined'=精细模式，'fast'=快速模式）
- 返回：换背景后的图像

**抠图算法**:
- 精细模式：使用 U2-Net 深度学习模型，边缘精细
- 快速模式：使用 GrabCut 算法，速度快

`python
def enhance_image(self, image, brightness, contrast, saturation, sharpness, gamma)
`
图像增强：
- rightness: 亮度调整（-100 ~ +100）
- contrast: 对比度调整（-100 ~ +100）
- saturation: 饱和度调整（-100 ~ +100）
- sharpness: 锐化强度（0 ~ 100）
- gamma: 伽马校正（0.5 ~ 2.0）

`python
def resize_to_spec(self, image, spec_name, dpi=300)
`
调整图像到指定规格尺寸：
- 根据规格名称获取目标尺寸（毫米）
- 根据DPI计算像素尺寸
- 使用高质量插值算法调整大小

#### 3.2.2 AIProcessor 控制器

**主要功能**:

9. 预览处理结果
10. 如果不满意，调整参数重新处理
11. 满意后点击"保存照片"
12. 保存到数据库并更新用户状态

### 3.4 技术要点

#### 3.4.1 智能裁剪算法
```python
# 1. 检测人脸位置
face_box = ai_processor.detect_face(image)
face_center_x = (face_box[0] + face_box[2]) / 2
face_center_y = (face_box[1] + face_box[3]) / 2

# 2. 根据规格计算裁剪区域
spec = PHOTO_SPECS[spec_name]  # 获取规格（宽x高，单位mm）
aspect_ratio = spec['width'] / spec['height']

# 3. 人脸居中，头顶留白30%
crop_height = (face_box[3] - face_box[1]) * 3.5
crop_width = crop_height * aspect_ratio
crop_x = face_center_x - crop_width / 2
crop_y = face_box[1] - crop_height * 0.3
```

#### 3.4.2 背景替换技术
**精细模式（U2-Net）**:
- 深度学习模型，专门用于显著性目标检测
- 输入：320x320 RGB图像
- 输出：人像掩码（0-255灰度图）
- 优点：边缘精细，头发丝清晰
- 缺点：耗时1-2秒

**快速模式（GrabCut）**:
- OpenCV传统算法
- 基于图割理论的前景分割
- 需要提供前景区域提示（人脸框）
- 优点：速度快（0.2秒）
- 缺点：边缘不够精细

#### 3.4.3 美颜算法（GFPGAN）
GFPGAN（Generative Facial Prior GAN）是腾讯开源的人脸修复模型：
- 基于GAN的生成式模型
- 可以修复模糊、低质量人脸
- 同时具有美颜效果（磨皮、提亮）
- 强度参数控制原图和美颜图的混合比例

```python
# 美颜处理
restored_face = gfpgan.enhance(face_image)
# 混合原图和美颜图
result = cv2.addWeighted(face_image, 1-strength, restored_face, strength, 0)
```


**处理流程**:
```python
for image_item in image_queue:
    # 1. 加载图像
    image = cv2.imread(image_item['input_path'])
    
    # 2. 应用处理参数
    processed = processor.crop_photo(image, params['crop_spec'])
```

## 7. 辅助功能模块

### 7.1 重复采集检查（DuplicateChecker）

#### 7.1.1 模块概述
防止同一用户重复拍照，通过人脸特征比对识别重复采集。

#### 7.1.2 核心功能
**文件路径**: `controllers/duplicate_checker.py`

**关键方法**:
```python
def check_duplicate_for_user(self, user_id: int, image: np.ndarray) -> Dict
```
检查用户是否已采集过照片：
- 从数据库读取用户已保存的人脸特征
- 提取新照片的人脸特征
- 计算特征向量的欧氏距离
- 相似度 > 60% 判定为重复

**业务流程**:
```
拍照 → 提取人脸特征 → 与数据库特征比对 → 判断是否重复 → 提示用户
```

**技术要点**:
- 使用128维人脸特征向量
- 特征存储在数据库的 `face_encoding` 字段（BLOB类型）
- 首次采集时保存特征，后续采集时比对

### 10.1 自动拍照功能

#### 10.1.1 功能说明
当启用自动拍照后，系统会实时监测拍照条件，当所有条件满足时自动倒计时3秒后拍照，无需手动点击按钮。

#### 10.1.2 触发条件
```python
is_perfect = score >= 70 and similarity >= 0.6
```
- 照片质量评分 ≥ 70分
- 与身份证照片相似度 ≥ 60%
- 检测到且仅有1个人脸

#### 10.1.3 工作流程
```
启用自动拍照 → 实时检测状态 → 状态完美 → 语音提示"3秒后自动拍照" 
→ 启动倒计时 → 每秒更新显示 → 倒计时归零 → 自动拍照 → 语音提示"拍照完成"
```

**状态改变处理**:
- 如果倒计时期间状态变为不完美（质量下降或相似度降低）
- 立即取消倒计时
- 语音提示："状态改变，已取消自动拍照"
- 当状态再次完美时，重新开始倒计时

#### 10.1.4 实现代码
```python
def _check_auto_photo(self, score: float, similarity: float):
    """检查是否应该自动拍照"""
    if not self.auto_photo_checkbox.isChecked():
        return  # 未启用自动拍照
    
    is_perfect = score >= 70 and similarity >= 0.6
    
    if is_perfect:
        if self.auto_photo_countdown == 0:
            self.auto_photo_countdown = 3
            self.voice_helper.speak("状态完美，3秒后自动拍照")
            self.auto_photo_timer.start(1000)  # 每秒触发
    else:
        if self.auto_photo_countdown > 0:
            self.auto_photo_countdown = 0
            self.auto_photo_timer.stop()
            self.voice_helper.speak("状态改变，已取消自动拍照")

def _auto_photo_countdown(self):
    """倒计时处理"""
    self.auto_photo_countdown -= 1
    
    if self.auto_photo_countdown > 0:
        # 更新显示
        self.quality_label.setText(f"自动拍照倒计时: {self.auto_photo_countdown}秒")
    else:
        # 倒计时结束，执行拍照
        self.auto_photo_timer.stop()
        self.take_photo()
        self.voice_helper.speak("拍照完成")
```

#### 10.1.5 用户体验优化
- 倒计时数字实时显示在质量标签上
- 语音提示倒计时开始和结束
- 状态改变时立即取消，避免误拍
- 拍照完成后自动重置，可以继续下一次自动拍照

### 15.4 身份证验证器（IDCardValidator）

#### 15.4.1 功能说明

#### 15.4.2 验证规则
**18位身份证号码结构**:
- 前6位：地区码
- 第7-14位：出生日期（YYYYMMDD）
- 第15-17位：顺序码（奇数为男，偶数为女）
- 第18位：校验码

**校验算法**:
```python
weights = [7, 9, 10, 5, 8, 4, 2, 1, 6, 3, 7, 9, 10, 5, 8, 4, 2]
check_codes = '10X98765432'

# 计算加权和
total = sum(int(id_number[i]) * weights[i] for i in range(17))
# 取模得到校验码
check_code = check_codes[total % 11]
# 验证最后一位
return id_number[17] == check_code
```

#### 15.4.3 验证内容
- 长度检查（必须18位）
- 格式检查（前17位为数字，最后一位为数字或X）
- 地区码检查（前6位是否有效）
- 出生日期检查（是否为有效日期）
- 校验码检查（使用加权算法）
