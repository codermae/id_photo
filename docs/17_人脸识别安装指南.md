## 问题分析

### 为什么 dlib 安装失败？

dlib 是一个 C++ 库，需要编译才能使用。编译需要：
- **C++ 编译器** - Visual Studio Build Tools 或 MinGW
- **CMake** - 构建工具

### 为什么不能用其他库？

根据论文要求，项目必须使用：
- **face_recognition** - 基于 dlib 的深度学习库
- **dlib CNN** - 用于人脸检测和特征提取
- **MediaPipe** - 用于人脸关键点检测

这些都是业界标准的深度学习库，符合"基于深度学习的图像处理模块"的要求。

## 技术架构

### 人脸识别流程

```
输入图像
   ↓
FaceRecognitionManager
   ├─ 检查 face_recognition 是否可用
   │  ├─ 是 → 使用 dlib CNN (高精度)
   │  └─ 否 → 使用 OpenCV Haar Cascade (备选)
   ↓
人脸检测 (face_locations)
   ↓
人脸特征提取 (encode_face)
   ↓
特征向量 (128维)
   ↓
保存到数据库
```

### 防重复采集流程

```
新拍摄照片
   ↓
提取人脸特征
   ↓
与数据库中的特征比对
   ↓
计算相似度
   ↓
判断是否重复
   ├─ 相似度 > 60% → 重复
   └─ 相似度 ≤ 60% → 新用户
```

## 故障排除

### 问题 1: CMake 安装后仍然报错

**解决方案:**
- 重启 PowerShell/CMD 窗口
- 运行 `cmake --version` 验证
- 确保 CMake 已添加到 PATH

### 问题 2: dlib 编译失败

**解决方案:**
- 确保已安装 Visual Studio Build Tools
- 或从 https://cmake.org 下载官方 CMake
- 不要使用 pip 安装的 cmake

### 问题 3: face_recognition 导入失败

**解决方案:**
- 确保 dlib 已成功安装
- 运行 `pip install --upgrade face-recognition`
- 检查 Python 版本: `python --version`

### 问题 4: 人脸识别精度低

**解决方案:**
- 确保使用的是 face_recognition (dlib) 而不是 OpenCV
- 检查 `system_status.py` 的输出
- 确保照片质量良好（光线充足，人脸清晰）
