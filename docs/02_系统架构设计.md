# 02 系统架构设计

## 2.1 整体架构

系统采用经典的**MVC（Model-View-Controller）架构**，实现了良好的模块化和低耦合。

```
┌─────────────────────────────────────────────────────────┐
│                      Presentation Layer                  │
│                    (views/ - PyQt5界面)                  │
├─────────────────────────────────────────────────────────┤
│                      Business Layer                       │
│              (controllers/ - 业务逻辑控制器)              │
├─────────────────────────────────────────────────────────┤
│                        Data Layer                         │
│          (models/ + utils/ - 数据模型和工具类)           │
├─────────────────────────────────────────────────────────┤
│                      Database Layer                       │
│                  (SQLite + SQLAlchemy)                    │
└─────────────────────────────────────────────────────────┘
```

## 2.3 技术栈详解

### 2.3.1 核心框架

#### PyQt5 (界面框架)
**选型理由**:
- 跨平台支持（Windows/Linux/Mac）
- 功能强大，组件丰富
- 信号槽机制，事件处理优雅
- 支持多线程，性能好

**主要使用**:
- QMainWindow: 主窗口
- QTabWidget: 标签页
- QTableWidget: 数据表格
- QWebEngineView: 图表显示
- QThread: 多线程处理

#### SQLAlchemy (ORM框架)
**选型理由**:
- 对象关系映射，代码简洁
- 支持多种数据库
- 自动管理连接池
- 事务管理完善

**数据库选择**: SQLite
- 轻量级，无需安装
- 单文件存储，易备份
- 性能满足需求
- 适合桌面应用

### 2.3.2 AI技术栈

#### MediaPipe (人脸检测)
**版本**: 0.10.x  
**用途**: 实时人脸检测和关键点定位  
**优势**:
- Google开源，质量可靠
- CPU运行，无需GPU
- 检测速度快（< 100ms）
- 精度高（> 99%）

#### GFPGAN (AI美颜)
**版本**: 1.3.x  
**用途**: 人脸增强和美颜  
**优势**:
- 基于GAN的深度学习模型
- 效果自然，不失真
- 支持参数调节
- 开源免费

#### OpenCV (图像处理)
**版本**: 4.x  
**用途**: 基础图像处理  
**功能**:
- 图像读取/保存
- 颜色空间转换
- 图像缩放/裁剪
- 滤波/增强

### 2.3.3 数据可视化

#### ECharts
**版本**: 5.x  
**集成方式**: QWebEngineView + CDN  
**优势**:
- 交互性强（悬停/缩放/平移）
- 图表类型丰富
- 配置灵活
- 性能优秀

## 2.5 数据流向

### 2.5.1 用户采集流程
```
身份证读取/Excel导入
    ↓
用户信息存入数据库 (User表)
    ↓
摄像头拍照
    ↓
原始照片保存 + 记录 (Photo表)
    ↓
AI处理（美颜/背景/裁剪）
    ↓
处理照片保存 + 记录 (Photo表)
    ↓
更新采集记录 (CollectionRecord表)
```

### 2.5.2 批量处理流程
```
选择用户/文件夹
    ↓
创建批量任务
    ↓
QThread后台处理
    ↓  (信号)
进度更新 → UI显示
    ↓
处理完成 → 保存结果
```

### 2.5.3 统计报表流程
```
选择筛选条件
    ↓
DataWorker(QThread)查询数据
    ↓
计算统计指标
    ↓
生成ECharts配置
    ↓  (信号)
ChartWidget显示图表
```

## 2.6 线程模型

### 2.6.1 主线程
**职责**: UI渲染和用户交互  
**原则**: 不执行耗时操作

### 2.6.2 工作线程
**类型**: QThread子类  
**用途**:
- 批量图像处理
- 数据库查询统计
- AI模型推理
- 文件IO操作

**通信机制**: 信号槽（Signal-Slot）
```python
class DataWorker(QThread):
    data_ready = pyqtSignal(dict)  # 定义信号
    
    def run(self):
        # 后台处理
        result = self.process_data()
        self.data_ready.emit(result)  # 发送信号

# 主线程连接信号
worker = DataWorker()
worker.data_ready.connect(self.update_ui)  # 连接槽函数
worker.start()
```

### 2.6.3 Chromium进程
**用途**: ECharts图表渲染  
**组件**: QWebEngineView  
**优势**: 独立进程，不影响主线程

## 2.7 设计模式

### 2.7.1 MVC模式
- **Model**: models/ 目录，数据模型
- **View**: views/ 目录，界面视图
- **Controller**: controllers/ 目录，业务逻辑

### 2.7.2 单例模式
**应用**: DatabaseHelper  
**目的**: 全局唯一数据库连接

### 2.7.3 工厂模式
**应用**: 图表生成器  
**目的**: 统一创建不同类型图表

### 2.7.4 观察者模式
**应用**: Qt信号槽机制  
**目的**: 解耦组件间通信
