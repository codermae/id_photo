# 08 部署运维指南

## 8.1 环境要求

### 8.1.1 硬件要求
- **CPU**: Intel i5或以上（支持AVX指令集）
- **内存**: 8GB以上
- **硬盘**: 10GB可用空间
- **摄像头**: USB摄像头（可选）
- **读卡器**: 二代身份证读卡器（可选）

### 8.1.2 软件要求
- **操作系统**: Windows 10/11 (64位)
- **Python**: 3.8 - 3.11
- **显卡**: 无需独立显卡（CPU运行）

## 8.2 安装步骤

### 8.2.1 安装Python

1. 下载Python 3.8+: https://www.python.org/downloads/
2. 安装时勾选"Add Python to PATH"
3. 验证安装:
```bash
python --version
pip --version
```

### 8.2.2 安装依赖

```bash
cd id_photo_system
pip install -r requirements.txt
```

**主要依赖**:
```
PyQt5==5.15.11
PyQtWebEngine==5.15.7
opencv-python==4.8.1
mediapipe==0.10.8
gfpgan==1.3.8
SQLAlchemy==2.0.23
pandas==2.1.3
openpyxl==3.1.2
```

### 8.2.3 下载AI模型

**GFPGAN模型**:
1. 下载地址: https://github.com/TencentARC/GFPGAN/releases
2. 下载文件: GFPGANv1.3.pth (约350MB)
3. 放置位置: `gfpgan/weights/GFPGANv1.3.pth`

### 8.2.4 初始化数据库

首次运行会自动创建数据库:
```bash
python main.py
```

数据库文件位置: `data/database/id_photo.db`

## 8.3 配置说明

### 8.3.1 系统配置 (config/config.py)

```python
# 照片存储路径
PHOTO_DIR = 'data/photos'

# 数据库路径
DATABASE_PATH = 'data/database/id_photo.db'

# AI模型路径
GFPGAN_MODEL_PATH = 'gfpgan/weights/GFPGANv1.3.pth'

# 默认美颜强度
DEFAULT_BEAUTIFY_STRENGTH = 0.8

# 照片质量
PHOTO_QUALITY = 95  # JPEG质量 (0-100)
```

### 8.3.2 数据库配置 (config/database.py)

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

DATABASE_URL = f"sqlite:///{DATABASE_PATH}"

engine = create_engine(
    DATABASE_URL,
    echo=False,  # 是否打印SQL语句
    pool_pre_ping=True,
    pool_recycle=3600
)

SessionLocal = sessionmaker(bind=engine)
```

## 8.5 备份恢复

### 8.5.1 数据备份

**自动备份**:
- 位置: `data/backups/`
- 命名: `id_photo_backup_YYYYMMDD_HHMMSS.db`

**手动备份**:
1. 点击"备份恢复"按钮
2. 选择"创建备份"
3. 输入备份说明
4. 确认备份

### 8.5.2 数据恢复

1. 点击"备份恢复"按钮
2. 选择备份文件
3. 点击"恢复"
4. 确认恢复（会备份当前数据）

### 8.5.3 照片备份

照片文件位置: `data/photos/`

建议定期备份整个`data`目录。

## 8.6 常见问题

### 8.6.1 摄像头无法打开

**原因**: 摄像头被其他程序占用或驱动问题

**解决**:
1. 关闭其他使用摄像头的程序
2. 重新插拔摄像头
3. 更新摄像头驱动
4. 在系统设置中检查摄像头权限

### 8.6.2 身份证读卡器无法识别

**原因**: 驱动未安装或读卡器故障

**解决**:
1. 安装读卡器驱动程序
2. 检查USB连接
3. 更换USB接口
4. 联系读卡器厂商

### 8.6.3 AI美颜速度慢

**原因**: CPU性能不足或模型未正确加载

**解决**:
1. 关闭其他占用CPU的程序
2. 检查GFPGAN模型文件是否存在
3. 降低美颜强度
4. 考虑升级硬件

### 8.6.4 图表不显示

**原因**: 网络问题或QWebEngineView未安装

**解决**:
1. 检查网络连接（ECharts从CDN加载）
2. 安装PyQtWebEngine: `pip install PyQtWebEngine`
3. 重启程序

### 8.6.5 数据库锁定

**原因**: 多个程序同时访问数据库

**解决**:
1. 关闭其他打开数据库的程序
2. 重启系统
3. 删除数据库锁文件（如有）

## 8.7 性能优化

### 8.7.1 系统优化

**关闭不必要的程序**:
- 释放内存和CPU资源
- 提高处理速度

**定期清理**:
- 删除过期照片
- 清理数据库（VACUUM）
- 清理临时文件

### 8.7.2 数据库优化

**定期维护**:
```sql
-- 清理碎片
VACUUM;

-- 分析统计信息
ANALYZE;

-- 重建索引
REINDEX;
```

**查询优化**:
- 使用索引
- 避免全表扫描
- 合理使用JOIN

### 8.7.3 照片存储优化

**压缩策略**:
- JPEG质量设置为95
- 大图自动压缩
- 定期清理重复照片

**目录结构**:
```
data/photos/
├── raw/          # 原始照片
│   ├── 2024/
│   │   ├── 09/
│   │   │   ├── user_1_raw_20240901.jpg
│   │   │   └── ...
└── processed/    # 处理后照片
    ├── 2024/
    │   ├── 09/
    │   │   ├── user_1_一寸_white.jpg
    │   │   └── ...
```

## 8.8 日志管理

### 8.8.1 日志位置

- 应用日志: `logs/app.log`
- 错误日志: `logs/error.log`

### 8.8.2 日志级别

- DEBUG: 调试信息
- INFO: 一般信息
- WARNING: 警告信息
- ERROR: 错误信息
- CRITICAL: 严重错误

### 8.8.3 日志配置

```python
import logging

logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    handlers=[
        logging.FileHandler('logs/app.log'),
        logging.StreamHandler()
    ]
)
```
