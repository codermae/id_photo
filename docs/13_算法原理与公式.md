## 重复采集检测算法

### 1.1 算法原理

重复采集检测算法通过人脸特征编码和相似度计算来判断用户是否已采集过照片。核心思想是将人脸图像转换为高维特征向量，然后通过计算两个特征向量之间的距离来判断是否为同一个人。

### 1.2 核心公式

#### 1.2.1 欧几里得距离（Euclidean Distance）

$$d = \sqrt{\sum_{i=1}^{n} (x_i - y_i)^2}$$

其中：
- $x_i, y_i$ 为两个人脸特征向量的第 $i$ 个分量
- $n$ 为特征向量的维度（通常为 128 维）
- $d$ 为两个特征向量之间的欧几里得距离

#### 1.2.2 相似度计算

$$\text{Similarity} = 1 - \min\left(\frac{d}{2}, 1\right)$$

其中：
- $d$ 为欧几里得距离
- 相似度范围为 $[0, 1]$，1 表示完全相同，0 表示完全不同
- 除以 2 是为了将距离归一化到 $[0, 1]$ 范围

#### 1.2.3 重复判断

$$\text{IsDuplicate} = \begin{cases}
\text{True} & \text{if } \text{Similarity} \geq \text{threshold} \\
\text{False} & \text{otherwise}
\end{cases}$$

其中：
- $\text{threshold}$ 为相似度阈值，默认为 0.6
- 当相似度大于等于阈值时，判定为重复采集

### 1.3 算法流程

```
输入：新拍摄的人脸图像
  ↓
1. 人脸检测与特征提取
   - 使用 dlib 或 face_recognition 库提取人脸特征
   - 生成 128 维特征向量 E_new
  ↓
2. 从数据库获取已存储的特征
   - 查询用户的历史人脸特征 E_old
  ↓
3. 计算特征距离
   - 计算欧几里得距离：d = ||E_new - E_old||
  ↓
4. 计算相似度
   - Similarity = 1 - min(d/2, 1)
  ↓
5. 判断是否重复
   - if Similarity >= 0.6: 重复采集
   - else: 新采集
  ↓
输出：重复检测结果
```

### 1.4 参数说明

| 参数 | 默认值 | 范围 | 说明 |
|---|---|---|---|
| 相似度阈值 | 0.6 | 0.5-0.8 | 越高越严格，越低越宽松 |
| 特征维度 | 128 | 固定 | dlib 人脸识别的特征维度 |
| 距离归一化系数 | 2.0 | 固定 | 用于将距离映射到 [0,1] |

### 1.5 性能指标

- **准确率**: 99%+ （在标准数据集上）
- **误识率**: < 0.1% （不同人被识别为同一人）
- **漏识率**: < 1% （同一人被识别为不同人）
- **处理速度**: ~50ms per image

### 1.6 代码实现

**位置**: `utils/face_recognition_manager.py` (第92-115行)

## 智能裁剪算法

### 2.1 算法原理

智能裁剪算法基于人脸检测结果，自动计算最优的裁剪区域，确保人物居中且符合证件照标准。算法考虑了头发、头顶空间、下巴空间等因素，生成符合国际标准的证件照。

### 2.2 核心公式

#### 2.2.1 完整头部高度估算

$$H_{\text{full}} = H_{\text{face}} + H_{\text{hair}} + M_{\text{top}} + M_{\text{bottom}}$$

其中：
- $H_{\text{face}}$ 为人脸检测框的高度
- $H_{\text{hair}}$ 为估算的头发高度，通常为 $0.4 \times H_{\text{face}}$
- $M_{\text{top}}$ 为头顶留白，通常为 $0.35 \times H_{\text{full}}$
- $M_{\text{bottom}}$ 为下巴留白，通常为 $0.25 \times H_{\text{full}}$

#### 2.2.2 照片尺寸计算

$$W_{\text{photo}} = H_{\text{photo}} \times \text{AspectRatio}$$

其中：
- $H_{\text{photo}}$ 为照片高度
- $\text{AspectRatio}$ 为宽高比，例如一寸照片为 $\frac{25}{35} \approx 0.714$

#### 2.2.3 裁剪区域计算

$$\begin{cases}
x_{\text{crop}} = \max(0, \min(w - W_{\text{crop}}, x_{\text{center}} - \frac{W_{\text{crop}}}{2})) \\
y_{\text{crop}} = \max(0, H_{\text{full\_top}} - M_{\text{top}})
\end{cases}$$

其中：
- $x_{\text{center}}$ 为人脸中心 x 坐标
- $H_{\text{full\_top}}$ 为完整头部的顶部 y 坐标
- $(x_{\text{crop}}, y_{\text{crop}})$ 为裁剪区域的左上角坐标

#### 2.2.4 缩放因子计算

$$\text{Scale} = \min\left(\frac{W_{\text{target}}}{W_{\text{crop}}}, \frac{H_{\text{target}}}{H_{\text{crop}}}\right)$$

其中：
- $W_{\text{target}}, H_{\text{target}}$ 为目标尺寸（例如 590×826 像素）
- $W_{\text{crop}}, H_{\text{crop}}$ 为裁剪后的尺寸

### 2.3 算法流程

```
输入：原始图像，目标规格（如"一寸"）
  ↓
1. 人脸检测
   - 检测人脸位置 (x, y, w, h)
  ↓
2. 估算完整头部
   - H_hair = 0.4 × h
   - H_full = h + H_hair + M_top + M_bottom
  ↓
3. 计算照片尺寸
   - W_photo = H_photo × AspectRatio
  ↓
4. 计算裁剪区域
   - x_crop = center_x - W_crop/2
   - y_crop = full_top - M_top
  ↓
5. 执行裁剪
   - cropped = image[y_crop:y_crop+H_crop, x_crop:x_crop+W_crop]
  ↓
6. 调整到目标尺寸
   - resized = resize(cropped, (W_target, H_target))
  ↓
输出：符合规格的证件照
```

### 2.4 证件照规格标准

| 规格 | 尺寸(mm) | 像素(600DPI) | 宽高比 | 用途 |
|---|---|---|---|---|
| 一寸 | 25×35 | 590×826 | 0.714 | 身份证、学生证 |
| 小二寸 | 35×49 | 826×1158 | 0.758 | 护照、港澳通行证 |
| 二寸 | 35×53 | 826×1252 | 0.660 | 签证、毕业证 |
| 美国护照 | 51×51 | 1200×1200 | 1.000 | 美国护照 |
| 欧盟护照 | 35×45 | 826×1063 | 0.777 | 欧盟护照 |

### 2.5 参数说明

| 参数 | 默认值 | 说明 |
|---|---|---|
| 头发高度比例 | 0.40 | 相对于人脸高度 |
| 头顶留白比例 | 0.35 | 相对于完整头部高度 |
| 下巴留白比例 | 0.25 | 相对于完整头部高度 |

## 磨皮美颜算法

### 3.1 算法原理

磨皮美颜算法通过双边滤波（Bilateral Filter）对皮肤进行平滑处理，同时保护眼睛、眉毛、嘴巴等重要特征。算法的核心是在平滑皮肤的同时保持边界清晰。

### 3.2 核心公式

#### 3.2.1 双边滤波（Bilateral Filter）

$$BF(x) = \frac{1}{W_p} \sum_{x_i \in \Omega} f(x_i) \cdot g_s(||x - x_i||) \cdot g_r(|I(x) - I(x_i)|)$$

其中：
- $f(x_i)$ 为像素值
- $g_s$ 为空间高斯核：$g_s(d) = e^{-\frac{d^2}{2\sigma_s^2}}$
- $g_r$ 为值域高斯核：$g_r(d) = e^{-\frac{d^2}{2\sigma_r^2}}$
- $\sigma_s$ 为空间标准差（控制平滑范围）
- $\sigma_r$ 为值域标准差（控制颜色相似度）
- $W_p$ 为归一化系数
- $\Omega$ 为滤波窗口

#### 3.2.2 磨皮强度参数

$$\begin{cases}
d = \max(9, 15 + 20 \times \text{strength}) \\
\sigma_{\text{color}} = \max(30, 40 + 60 \times \text{strength}) \\
\sigma_{\text{space}} = \max(30, 40 + 60 \times \text{strength}) \\
\text{iterations} = \max(1, 1 + \text{strength} \times 2)
\end{cases}$$

其中：
- $d$ 为滤波核大小（9-35 像素）
- $\sigma_{\text{color}}$ 为颜色标准差（30-100）
- $\sigma_{\text{space}}$ 为空间标准差（30-100）
- $\text{strength}$ 为磨皮强度（0-1）
- $\text{iterations}$ 为迭代次数（1-3 次）

#### 3.2.3 特征保护掩膜

$$\text{ProtectMask} = \text{EyeMask} \cup \text{EyebrowMask} \cup \text{MouthMask} \cup \text{NoseMask}$$

其中：
- 眼睛区域：半径 15 像素的圆形
- 眉毛区域：椭圆形（20×10 像素）
- 嘴巴区域：椭圆形（20×10 像素）
- 鼻孔区域：半径 8 像素的圆形

#### 3.2.4 混合公式

$$\text{Result}(x, y, c) = \text{Image}(x, y, c) \times (1 - \text{mask}(x, y) \times \text{blend\_strength}) + \text{Smoothed}(x, y, c) \times \text{mask}(x, y) \times \text{blend\_strength}$$

其中：
- $\text{mask}(x, y)$ 为皮肤区域掩膜（0-1）
- $\text{blend\_strength}$ 为混合强度（0-0.8）
- $c$ 为颜色通道（B、G、R）

### 3.3 祛痘算法

#### 3.3.1 痘痘检测

使用形态学操作检测痘痘：

$$\text{TopHat} = \text{Image} - \text{Opening}(\text{Image})$$
$$\text{BlackHat} = \text{Closing}(\text{Image}) - \text{Image}$$
$$\text{Combined} = \text{TopHat} + \text{BlackHat}$$

其中：
- Opening 为先腐蚀后膨胀
- Closing 为先膨胀后腐蚀
- 核大小为 9×9 像素

#### 3.3.2 痘痘大小过滤

$$\text{BlemishArea} = \begin{cases}
\text{True} & \text{if } 2 \leq \text{Area} \leq 50 + 100 \times \text{strength} \\
\text{False} & \text{otherwise}
\end{cases}$$

其中：
- 最小面积：2 像素
- 最大面积：50-150 像素（取决于强度）

#### 3.3.3 圆度检测

$$\text{Circularity} = \frac{4\pi \times \text{Area}}{\text{Perimeter}^2}$$

只保留圆度 > 0.3 的区域（痘痘通常比较圆）

#### 3.3.4 Inpaint 修复

$$\text{Result} = \text{Inpaint}(\text{Image}, \text{BlemishMask}, \text{radius})$$

其中：
- $\text{radius}$ 为修复半径（5-10 像素）
- 使用 Telea 算法进行修复

### 3.4 眼部增强算法

#### 3.4.1 LAB 色彩空间增强

$$\begin{cases}
L' = L + \text{strength} \times 10 \\
a' = a \\
b' = b
\end{cases}$$

其中：
- $L$ 为亮度通道
- $a, b$ 为色度通道
- 增强亮度使眼睛更明亮

#### 3.4.2 对比度增强

$$L' = L \times (1 + \text{strength} \times 0.3)$$

### 3.5 唇部增强算法

#### 3.5.1 红色通道增强

$$R' = R + \text{mask}(x, y) \times \text{strength} \times 30$$

其中：
- 只增强嘴巴区域的红色通道
- 增强量为 0-30（取决于强度）

### 3.6 牙齿美白算法

#### 3.6.1 LAB 空间美白

$$L' = L + \text{mask}(x, y) \times 20$$

其中：
- 只增加牙齿区域的亮度
- 增加量为 20

### 3.7 算法流程

```
输入：原始图像，美颜选项，强度设置
  ↓
1. 人脸检测与特征提取
   - 检测人脸区域
   - 估算关键点位置
  ↓

   - 形态学检测痘痘

  ↓

   - LAB 空间增强亮度
   - 增加对比度
  ↓
6. 唇部增强（如果启用）
   - 增强红色通道
  ↓
7. 牙齿美白（如果启用）
   - 增加亮度
  ↓
8. 其他增强（瘦脸、大眼等）

  ↓
输出：美颜处理后的图像
```

## 照片质量评分算法

### 4.1 算法原理

照片质量评分算法通过多维度评估照片质量，包括清晰度、亮度、对比度等，最终生成综合质量评分（0-100）。该算法能够快速判断照片是否满足证件照的质量要求。

### 4.2 核心公式

#### 4.2.1 清晰度评分（基于拉普拉斯方差）

$\text{Sharpness} = \min\left(100, \left\lfloor \frac{\text{Laplacian\_Variance}}{5} \right\rfloor\right)$

其中：
- $\text{Laplacian\_Variance}$ 为拉普拉斯算子的方差
- 拉普拉斯算子用于检测图像的边缘变化
- 方差越大，边缘变化越明显，图像越清晰
- 最大值为 100 分

#### 4.2.2 亮度评分（分段函数）

$\text{Brightness} = \begin{cases}
100 & \text{if } 100 \leq B \leq 150 \\
85 & \text{if } 80 \leq B \leq 170 \\
70 & \text{if } 60 \leq B \leq 190 \\
50 & \text{if } 40 \leq B \leq 210 \\
30 & \text{otherwise}
\end{cases}$

其中：
- $B$ 为灰度图像的平均亮度值（0-255）
- 最佳范围：100-150
- 过亮或过暗都会降分

#### 4.2.3 对比度评分（分段函数）

$\text{Contrast} = \begin{cases}
100 & \text{if } 60 \leq C \leq 80 \\
85 & \text{if } 50 \leq C \leq 90 \\
70 & \text{if } 40 \leq C \leq 100 \\
50 & \text{if } 30 \leq C \leq 110 \\
30 & \text{otherwise}
\end{cases}$

其中：
- $C$ 为灰度图像的标准差（衡量对比度）
- 最佳范围：60-80
- 过高或过低的对比度都会降分

#### 4.2.4 综合评分

$\text{Overall\_Score} = \left\lfloor 0.4 \times \text{Sharpness} + 0.3 \times \text{Brightness} + 0.3 \times \text{Contrast} \right\rfloor$

其中：
- 清晰度权重：40%（最重要）
- 亮度权重：30%
- 对比度权重：30%
- 综合评分范围：0-100

### 4.3 评分标准

| 评分范围 | 等级 | 说明 |
|---------|------|------|
| 90-100 | 优秀 | 照片质量完美，完全满足要求 |
| 80-89 | 良好 | 照片质量良好，可以接受 |
| 70-79 | 合格 | 照片质量合格，勉强满足要求 |
| < 70 | 不合格 | 照片质量不达标，需要重拍 |

### 4.4 算法流程

```
输入：照片图像
  ↓
1. 转换为灰度图
   - 将 BGR 图像转换为灰度
  ↓
2. 计算拉普拉斯方差
   - 应用拉普拉斯算子
   - 计算方差
   - 转换为清晰度评分
  ↓
3. 计算亮度指标
   - 计算平均像素值
   - 根据分段函数得到亮度评分
  ↓
4. 计算对比度指标
   - 计算像素值标准差
   - 根据分段函数得到对比度评分
  ↓
5. 计算综合评分
   - 加权求和：0.4 × 清晰度 + 0.3 × 亮度 + 0.3 × 对比度
  ↓
6. 返回详细评分
   - 综合评分、清晰度、亮度、对比度
  ↓
输出：质量评分字典
```

### 4.5 参数说明

| 参数 | 默认值 | 范围 | 说明 |

### 4.7 代码实现

# 清晰度检测（拉普拉斯方差）- 第356-358行
gray = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)
laplacian_var = cv2.Laplacian(gray, cv2.CV_64F).var()
sharpness_score = min(100, int(laplacian_var / 5))

# 亮度检测 - 第360-369行
brightness = np.mean(gray)
if 100 <= brightness <= 150:
    brightness_score = 100
elif 80 <= brightness <= 170:
    brightness_score = 85
elif 60 <= brightness <= 190:
    brightness_score = 70
elif 40 <= brightness <= 210:
    brightness_score = 50
else:
    brightness_score = 30

# 对比度检测 - 第371-380行
contrast = np.std(gray)
if 60 <= contrast <= 80:
    contrast_score = 100
elif 50 <= contrast <= 90:
    contrast_score = 85
elif 40 <= contrast <= 100:
    contrast_score = 70
elif 30 <= contrast <= 110:
    contrast_score = 50
else:
    contrast_score = 30

# 综合评分 - 第382行
overall_score = int(
    sharpness_score * 0.4 + 
    brightness_score * 0.3 + 
    contrast_score * 0.3
)
